# AppLovin — Old Friend Profile

## 这家公司发明了什么范式？

AppLovin发明的范式是**"算法是果，组织是因"**——在一个看似没有技术壁垒的广告中间商市场，通过极致精简的组织设计（约400人核心团队驱动超40亿美元EBITDA），将ML实验通量最大化，使算法迭代速度本身成为不可复制的竞争壁垒。

具体而言，AppLovin证明了三件事：
1. **效率组织 > 规模组织**：900人公司打败数万人的Meta和Google的网盟广告业务，人效超1800万美元（还原广告收入），是同行10倍以上
2. **闭环数据基建 + 极速迭代组织 = 算法飞轮**：通过MAX(SSP)+AppDiscovery(DSP)+Adjust(MMP)构建闭环数据三角，再用Pod制全栈工程师组织将"数据→实验→上线→反馈"的循环压缩到5天，竞争对手需要数月
3. **组织能力可跨域复用**：同一套"快速试错"的组织机器，从游戏广告迁移到电商广告，20-30人团队数月内ROAS逼近Meta 70-80%水平

## Fit Score — 5/5

AppLovin的组织形态与其业务本质达到了极致的适配：

效果广告的本质是**预测精度之争**——哪个平台ROAS更高，预算就流向哪里。预测精度取决于两个底层变量：数据质量和算法迭代速度。AppLovin的组织设计精准服务于这两个变量：

- **闭环数据三角**（MAX+AppDiscovery+Adjust）解决数据质量问题，且这个基建本身需要数年积累，构成时间壁垒
- **Pod制全栈工程师 + 反会议 + No PM + Revenue Impact考核**解决迭代速度问题，将"想法到上线"压缩到5天
- **极简代码架构**（3个Class支撑数十个模型）保证了组织规模不膨胀也能维持系统可理解性
- **CEO亲自控HC + 高淘汰率**保证了人才密度不因增长而稀释

这套组织形态不是模仿来的，而是从业务第一性原理推导出来的自然结果——当你的产品就是算法、你的竞争力就是迭代速度时，一切组织设计都应该服务于"让最强的工程师以最快速度迭代算法"这个单一目标。

## Step 0: 业务本质 → 组织形态推导

**业务本质**：效果广告的中间商，通过算法预测精度赚取广告主支出与流量成本之间的差价（take rate约35%）。核心竞争变量是ROAS——广告主预算流向ROAS最高的平台，Publisher流量流向eCPM最高的平台。

**理想组织形态推导**：
1. 算法效果 = f(数据质量, 迭代速度) → 需要闭环数据基建 + 极速实验组织
2. 算法迭代 = 大量ML实验并行 → 需要最小化跨人依赖 → Full-stack/Entire Stack工程师 + Pod制
3. 实验速度 = 消灭一切减速因素 → No PM + 反会议 + 极简代码架构 + 决策权集中
4. 组织不退化 = 人才密度不稀释 → HC极度克制 + 高淘汰 + AI替代人力 + Underdog文化
5. 数据飞轮自我强化 → 算法好→赢更多竞标→更多数据→算法更好 → 需要闭环基建（MAX+AppDiscovery+Adjust）

AppLovin的实际组织形态与这个推导结果高度一致，这就是Fit Score 5/5的原因。

## "学我者生，似我者死"边界

**可学的组织原则**（适用于算法/效率驱动型公司）：
- 极简架构思维：用最少的代码结构支撑最多的功能，降低系统认知成本
- Engineering-driven/去PM化：当核心竞争力是技术效果时，让最懂技术的人决定产品方向
- Revenue Impact考核：将工程师绩效与商业结果直接挂钩，消灭目标分裂
- 反会议/静默协作：制度化地消灭不必要的同步沟通
- HC克制 + AI增效：宁可用AI放大顶尖人才产出，也不用规模换增长

**不可学/学了会死的**：
- **"无PM"不等于"不需要产品思维"**：AppLovin能去PM化的前提是工程师本身具备极深的领域知识和商业判断力，且产品极度聚焦（一个核心产品）。如果公司有多条产品线、多类用户，盲目去PM化会导致产品方向混乱
- **"反会议"不等于"不需要对齐"**：AppLovin的静默协作建立在极简代码架构（3个Class）和闭环数据看板之上。如果系统架构复杂、信息不透明，去掉会议只会制造更多信息黑洞
- **"极致精简"不等于"少雇人"**：AppLovin的精简是高人才密度下的精简，每个人都是Entire Stack。如果团队能力不足就盲目缩编，只会导致关键环节无人负责
- **闭环数据三角不可复制**：MAX的SDK已装在15亿设备上，mediation份额55%，这是十年布局的结果。竞争对手即使有同等算法能力，也无法复制这个数据采集基础设施

### BI-1: AXON 2.0上线——从技术范式切换到收入爆发（2023年中）

| 层级 | 分析 |
|------|------|
| **A层（现象/数据）** | 2023年中AXON 2.0上线后：软件平台收入从2022年$11.7亿跃升至2023年$19.6亿（+68%），2024年$32.2亿（+64%），2025年$54.8亿（+70%）。EBITDA margin从2022年54%→2023年67%→2024年75%→2025年81%。游戏广告市占率达30%，iOS仅次于苹果，安卓仅次于Google，全面超越Meta |
| **B层（驱动机制）** | 技术范式切换：从监督学习到深度神经网络+Transformer；从身份图谱（IDFA依赖）到行为预测+上下文信号（后ATT原生）；模型按用户行为分类而非游戏品类。效果传导链：算法精度↑→广告主ROAS↑→CPM↑→收入和利润同时暴增。闭环数据三角使反馈延迟压缩到"几小时"而非"数周" |
| **C层（组织前因）** | 这不是技术创新驱动的inflection，而是**组织重构驱动的inflection**。"先换人再换技术"——2023年大规模替换老员工、引入Meta/Google新血、葛小川获得全部算法和产品决策权。AXON 2.0仅用11人团队从底层推翻重建。3个主Class支撑数十个模型的极简架构使新人onboarding成本极低 |
| **D层（时间差与投资含义）** | 组织换血（2023Q1）→ AXON 2.0上线（2023Q2-Q3）→ 收入爆发（2023Q3起）→ 股价反弹（2023年从$10涨至$50+）。**关键时间差：组织变化领先业绩约6个月，领先股价约3-6个月。如果在2023Q1观察到"大规模换血+葛小川升任CPEO"这两个组织信号，就能在股价$10-20时建仓，捕获后续70倍涨幅的起点** |

### BI-2: 电商广告从零到规模化验证（2023H2-2026）

### BI-3: MAX从推广期进入收割期（2018-2022推广，2023起收割）

| 层级 | 分析 |
|------|------|
| **A层（现象/数据）** | MAX在头部手游mediation份额达55%，是Unity LevelPlay的两倍以上。聚合超9,000款应用，SDK安装在15亿设备上。AppDiscovery在MAX平台竞标成功率超50%，免佣成本优势5%。广告主侧take rate达40-45%（行业平均20%）却仍提供更高ROAS |

| **C层（组织前因）** | Adam在2017-18年的战略认知超前性：当时市场共识是"中间商不可能成为giant"，Adam逆共识用最激进方式抢占基础设施层位置。BD团队KPI全部压在市占率而非短期盈利。2021年$10.5亿收购MoPub并关停——"消灭竞争对手+整合市场"的经典操作，将客户迁移至MAX |
| **D层（时间差与投资含义）** | 推广投入期（2017-2022，约5年）→ 收割期（2023年起）。**这是AppLovin最被低估的隐性资产——5年的基础设施投资在AXON 2.0爆发后一次性释放价值。MAX的垄断地位意味着竞争对手即使有同等算法能力，也无法复制这个数据采集基础设施层，因为SDK已经装在15亿设备上。市场长期将APP视为"中间商"而非"基础设施垄断者"，这个认知差在2023-2024年逐步修正，但可能仍未被充分定价** |

### BI-4: 苹果ATT政策——外部冲击创造的结构性机会（2021年4月）

| 层级 | 分析 |
|------|------|
| **A层（现象/数据）** | 2021年4月ATT实施，全球IDFA opt-in率仅~25%。Meta Audience Network因"双重授权"合规策略，iOS可触达受众接近归零，MAN于2022年退出市场（2026年才重新回来）。APP的AXON 1.0同样受创，2022年软件业务连续三季度收入持平 |
| **B层（驱动机制）** | ATT本质上重新洗牌了移动广告竞争格局：依赖IDFA身份图谱的玩家（Meta）受创最深，能够基于行为预测和上下文信号建模的玩家获得结构性优势。APP在短期内同样受损，但因其闭环数据三角（MAX+AppDiscovery+Adjust）不依赖IDFA，恢复速度远快于竞争对手 |
| **C层（组织前因）** | 2021年收购Adjust的时机精准——恰好在ATT实施前完成归因闭环。更关键的是，AXON 2.0的技术路线从出生第一天起就是为后ATT原生的——不依赖"你是谁"，而是预测"你此刻会做什么"。这一技术方向选择不是事后应对，而是组织换血后新团队的主动设计 |
| **D层（时间差与投资含义）** | ATT实施（2021.04）→ APP业绩低谷（2022全年）→ AXON 2.0上线反转（2023Q2-Q3）→ 股价从低点$10启动70倍涨幅。**关键教训：外部政策冲击对所有玩家造成短期伤害，但对拥有闭环数据基建的玩家是结构性利好。2022年全年是最佳买入窗口——市场线性外推短期业绩恶化，完全忽视了APP闭环数据三角在后ATT时代的结构性优势，以及正在进行的组织换血可能带来的技术范式切换** |

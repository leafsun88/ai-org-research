Collect 到 Analysis 的项目流程

当前项目只保留四个默认 source：podcast、financials、substack、youtube。通用脚本都在 scripts/discovery/，公司运行产物进入 companies/{slug}/_staging/，正式证据进入 companies/{slug}/vault/，中间研究层由 学习/skills/essence/SKILL.md 负责，分析只从 essence 和高可信原文开始。

配置与 skill 通用脚本 公司证据库 分析输出 归档，不默认调用

1. Target

公司配置

config/company_targets.json

定义 slug、aliases、founders、channels、output_paths。

↓

读配置层

scripts/discovery/target_config.py

统一解析 target、company root、_staging root、vault source 目录。

2. Collect Skill

采集入口

学习/skills/collect/SKILL.md

先写关键词宇宙，再跑四个 source，不再默认全网散抓。

↓

统一编排

scripts/discovery/collect_target.py

CLI 示例：python3 scripts/discovery/collect_target.py LOVABLE --dry-run

3. Generic Scripts

Podcast

fetch_podcasts.py
fetch_podcast_transcript.py

关键词、metadata、Groq / DashScope 转录、失败重试。

Financials

fetch_financials.py
fetch_sec_edgar.py
fetch_xbrl.py

上市公司财务、SEC、年报；私有公司默认跳过。

Substack / YouTube

fetch_web_sources.py
fetch_youtube.py

复用 podcast query map；YouTube 支持 RapidAPI 和免费 fallback。

4. Staging

运行区

companies/{slug}/_staging/sources/

只放脚本中间结果、status、failure、cache，不作为正式分析入口。

标准分层

{source}/metadata/
{source}/transcripts/

所有 source 都先在 staging 内分清 metadata 和正文。

5. Vault

正式证据库

companies/{slug}/vault/

每个 source 都拆成 metadata / transcripts / essence。

同步后材料

vault/{source}/metadata/
vault/{source}/transcripts/

raw transcript 只做核验，不作为最终分析的主要阅读入口。

6. Essence Skill

逐源提炼

学习/skills/essence/SKILL.md

逐段读材料，遇到重要机制、动作、数字、组织信号或业务信息就写一个 insight block。

研究中间层

vault/{source}/essence/

一份 source 对应一份 essence；后续分析默认先读这里。

7. Analysis

分析 skill

学习/skills/analysis/SKILL.md

吸收 alike 框架：业务本质、理想组织形态、Founder 认知、组织动作、业务传导、风险和可证伪点。

最终输出

companies/{slug}/analysis/

source inventory、evidence map、复杂报告、一页分析、500 字版本都落这里。

默认调用链

阶段 入口文件 读哪里 写哪里
关键词宇宙 fetch_podcasts.py --target TARGET config/company_targets.json _staging/sources/podcasts/metadata/_podcast_query_map.md
Podcast metadata fetch_podcasts.py Apple / RSS / query map _staging/sources/podcasts/metadata/
Podcast transcript fetch_podcast_transcript.py _staging/sources/podcasts/metadata/_podcast_episodes*.json _staging/sources/podcasts/transcripts/
YouTube fetch_youtube.py --target TARGET podcast query map + target manifest _staging/sources/youtube/{metadata,transcripts}/
Substack fetch_web_sources.py --target TARGET --channel substack podcast query map + target manifest _staging/sources/substack/{metadata,transcripts}/
Financials fetch_financials.py / SEC scripts ticker / SEC / yfinance _staging/sources/financials/{metadata,reports}/
Essence 学习/skills/essence/SKILL.md
build_transcript_essence_queue.py		 vault/{source}/transcripts/ / vault/financials/reports/ vault/{source}/essence/
Analysis 学习/skills/analysis/SKILL.md vault/*/essence/ + 必要原文 companies/{slug}/analysis/*.md

保留与精简

类型 位置 默认状态 原因
Active scripts scripts/discovery/*.py 保留 通用采集与迁移入口。
Active skills 学习/skills/collect/
学习/skills/analysis/		 保留 现在只需要采集和分析两条主链。
Optional sources vault/_optional/ 不默认调用 official / jobs / social / web 等信息熵高，只有用户点名再抓。
Legacy skills 学习/_legacy/skills/ 归档 保留旧思路，但不让 agent 默认误调用。
Legacy scripts / repos companies/_legacy/ 归档 不删除，保证可追溯；不进入当前 workflow。

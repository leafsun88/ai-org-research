# Palantir Technologies (PLTR) 组织架构穿透 — 2026-04-06

## 一、组织概览

Palantir Technologies成立于2003年，总部位于Denver, Colorado。由Peter Thiel、Alex Karp、Stephen Cohen、Joe Lonsdale和Nathan Gettings共同创立。2020年通过Direct Listing在NYSE上市，2022年转至Nasdaq。

**核心组织特征：**
- 三位创始人通过Class F股份的Founder Voting Trust控制49.99%投票权
- CEO Alex Karp是哲学博士（非传统科技CEO）
- "Forward Deployed Engineer"（FDE）模式 = 工程师驻扎客户现场
- 扁平但"painful"的内部结构
- 约4000+员工（远小于同等市值的科技公司）

## 三、关键观察

### 1. 创始人三角结构

**三位创始人的权力分工：**
- **Peter Thiel** (Chairman): 战略指导+政治关系+投资视野。不参与日常运营。
- **Alex Karp** (CEO): 公司的哲学灵魂+对外代言+战略方向。eccentric但极其有conviction。
- **Stephen Cohen** (President): 技术架构+产品方向。低调但权力极大。

Class F股份确保三人共同控制49.99%投票权 → 即使持股稀释，控制权永远不变。这是最极端的创始人保护结构之一。

### 2. CTO Shyam Sankar — FDE模式的创造者

- Employee #13，最早的非创始人核心成员
- 在反恐战争中"坐在帐篷里和美军一起"适配Palantir软件
- 创造了"Forward Deployed Engineer"模式 — 工程师直接驻扎客户现场
- 从一线工程师→管理全球部署→CTO → 公司第五位亿万富翁
- Ginkgo Bioworks Chairman = 外部视野

**FDE模式是Palantir组织DNA的核心：**
- 不是远程卖软件，而是派最优秀的工程师到客户一线
- 极高的客户理解 → 极高的产品适配度 → 极高的切换成本
- 但也 = 高人力成本 + 高burnout + 规模化挑战

### 3. CFO和CRO都有法律背景

- David Glazer (CFO): 律师出身 → 对合规/政府合同的理解
- Ryan Taylor (CRO+CLO): Stanford CS + Harvard Law → 兼任法务和销售 = 独特的合同谈判能力

这反映了Palantir业务的特殊性：政府合同需要极强的法律/合规能力，客户关系管理和合同谈判同样重要。

### 4. 无COO、无传统CMO

- 与Duolingo类似，无COO角色
- 没有传统的CMO → 依赖产品驱动+政府关系+bootcamp模式，而非传统营销

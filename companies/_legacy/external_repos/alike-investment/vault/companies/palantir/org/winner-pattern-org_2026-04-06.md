# Palantir Technologies — D1-D7 组织穿透

## Step 0: 业务本质画像

**Palantir在造什么？**
- 一个把AI部署到实际决策工作流的"操作系统"
- 不是卖数据工具，不是卖AI模型，而是帮组织在复杂环境中做决策并执行
- 客户是政府+军事+大型企业中最mission-critical的场景

**这要求什么样的组织？**
1. **极高的客户理解**：必须深入理解每个客户的业务逻辑才能映射到Ontology → 需要FDE模式
2. **极强的安全/合规能力**：政府+军事客户 → 需要法律背景的高管团队
3. **长期conviction**：Ontology的价值需要多年才能被客户理解 → 需要不受短期市场压力影响的创始人控制
4. **高人才密度+高强度文化**：mission-critical场景容错率极低 → 需要严格筛选+持续高压
5. **contrarian能力**：在市场不理解的时候坚持投入 → 需要哲学型领导者

**理想组织形态：** 高人才密度的扁平组织 + 创始人长期控制 + FDE模式深度嵌入客户 + 法律/合规基因内化

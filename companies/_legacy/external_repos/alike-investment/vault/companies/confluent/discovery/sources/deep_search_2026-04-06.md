# Confluent (CFLT) DeepSearch

## 公司概况与商业模式
- 由Apache Kafka联合创始人Jay Kreps创办，从managed Kafka演进为Data Streaming Platform (DSP)
- 两大产品线：Confluent Cloud（全托管SaaS）+ Confluent Platform（企业级自管理软件）
- 收入模式：消费制（consumption-based），客户按使用量付费
- FY2025总营收$1.17B，同比增长21%；其中订阅收入$1.12B占96%
- Confluent Cloud FY2025收入$624M，同比+27%，是主要增长引擎
- TAM从2021年~$50B增长到2025年超$100B

## 竞争格局
- 作为Kafka创始团队，对云厂商win rate >90%
- 主要竞争者：AWS Kinesis/MSK，Google Pub/Sub，Azure Event Hubs
- Snowflake在构建streaming-native架构，Salesforce收购Informatica加深数据层
- WarpStream（已被收购）的BYOC模式和Flink流处理扩展了竞争壁垒
- 被Forrester评为Streaming Data Platforms Q4 2025 Leader（14个标准最高分）

## CEO/创始人：Jay Kreps
- LinkedIn前数据基础设施首席架构师，Apache Kafka初始开发者
- 只上了一年高中（认为传统教育效率低），在UC Santa Cruz追回学业
- 管理哲学：承认CEO在"部分理解的迷雾中"运营；强调empathy；认为人被honor和recognition驱动
- 工程文化：工程团队自愿用最低title，认为在Confluent很少有人配得上"工程师"这个头衔
- 战略思维：all-in核心使命+失败，好过settle for a niche，这帮助Confluent全力投入云对抗AWS

## AI/技术战略
- Confluent Intelligence：扩展实时业务数据到企业AI
- Streaming Agents（2025.10发布）：构建event-driven AI agents，实时observe/reason/act
- Real-Time Context Engine：为AI提供实时上下文
- Tableflow：将Kafka topics表示为Apache Iceberg/Delta Lake表，连接data lake/warehouse
- Flink流处理：低八位数ARR，环比增长70%，FY2026将成为Cloud的实质贡献者
- WarpStream：diskless Kafka模型，8x消费增长，零数据丢失复制

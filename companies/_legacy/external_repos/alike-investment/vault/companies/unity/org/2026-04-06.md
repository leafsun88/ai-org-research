# Unity Software 组织 Winner Pattern 分析

## Step 0: 业务本质画像

**在造什么**：
- **输入**：游戏/应用开发者的创作需求 + 移动App的变现/用户获取需求
- **输出**：(1) Create: 实时3D引擎（Unity 6），让开发者构建跨平台游戏和应用；(2) Grow: AI驱动的广告变现/用户获取平台（Vector），帮开发者赚钱和获客
- **价值创造核心环节**：(1) 引擎生态锁定——全球70%+手游用Unity开发，开发者的技能投资和项目资产形成极高转换成本；(2) Create-to-Grow闭环——用引擎获取开发者，用广告平台变现，数据飞轮驱动广告效率；(3) AI重构广告竞价——Vector用AI替代传统waterfall，实现实时bidding优化

**环境特性**：
- **竞争格局**：引擎端双寡头（Unity vs Unreal），但Unreal聚焦AAA/高端，Unity统治移动端和indie。广告端面临AppLovin (AXON)、Google AdMob、Meta Audience Network的激烈竞争。Vector是直接对标AppLovin AXON的产品
- **反馈周期**：引擎业务长周期（版本迭代1-2年），广告业务短周期（实时bidding，天级/周级反馈）。两个业务本质不同，对组织的要求矛盾
- **稳定性**：中度不确定——移动广告市场受IDFA/隐私政策影响大（2021年ATT冲击），AI正在重塑整个广告技术栈
- **竞争性质**：引擎端接近双寡头稳态；广告端赢者通吃倾向强（数据飞轮+规模效应），AppLovin已证明AI-native广告平台可以获得80%+ margin

**理想组织形态**（从业务本质倒推）：

Unity的业务本质是"双引擎公司"——Create（工具平台/长周期）+ Grow（AI广告/短周期）。这要求：

| 业务特性 | 理想组织要求 |
|---------|------------|
| Create引擎需要长周期打磨+开发者信任 | 工程文化优先，产品稳定性>新功能速度；开发者关系团队需有真正的技术credibility |
| Grow广告需要AI-native快速迭代 | 小而精的ML团队，天级实验周期，数据驱动决策；类AppLovin的200人精英模式 |
| 两个业务本质冲突（长周期 vs 短周期） | 要么拆分为两个独立BU（各有适配的组织），要么找到一个CEO能同时驾驭两种节奏 |
| 2022-2024历史包袱极重（ironSource合并+Runtime Fee+CEO被炒） | 需要turnaround CEO：果断清理、重建信任、重新聚焦 |
| 广告平台规模效应极强 | 组织必须精简——人效是核心指标，bloat = 死亡 |

**Q4: 实际组织与理想组织的差距**：

Unity正处于turnaround的中段。Bromberg在清理历史包袱（关停ironSource Ads、剥离Supersonic、大规模裁员）上做得果断，但新组织形态尚未被验证。核心差距在于：(1) 人效$371K/人远低于AppLovin的$2M+/人，说明bloat尚未清除干净或业务效率未到位；(2) Vector广告平台虽增长强劲（YoY+72% best month），但尚未证明能达到AppLovin级别的margin和规模；(3) 开发者信任在Runtime Fee事件后严重受损，修复需要时间。

### D1: 创始人/CEO 认知与领导力 [3 / 5] — Turnaround CEO，非创始人，果断但尚未证明vision深度

**核心判断：Bromberg是合格的turnaround执行者，但不是vision型CEO。**

Matthew Bromberg 2024年5月接任，是Unity成立以来的第三任CEO（前两任：David Helgason创始人2014年交棒 -> John Riccitiello 2014-2023被董事会解雇 -> Jim Whitehurst interim -> Bromberg）。这一连串CEO更替本身就是组织不稳定的信号。

**1. Turnaround执行力——果断的"Burn the Boats"**

Bromberg上任后最重要的决策是关停legacy ironSource Ads Network（2026年4月30日），将广告业务all-in到Vector AI平台。这是教科书级的turnaround手法——砍掉仍在产生收入但战略方向错误的业务线，强迫组织向新方向迁移。同时探索剥离Supersonic游戏发行业务，进一步聚焦双支柱（Create + Grow/Vector）。

这种果断性值得认可。但"Burn the Boats"策略的风险在于：如果Vector无法在ironSource关停后快速接住收入，公司将面临严重的收入缺口。

**2. 背景适配度——mobile游戏+广告turnaround**

Bromberg的背景（Zynga COO参与turnaround、EA mobile高管、Major League Gaming CEO）与Unity的业务相当匹配——他理解mobile游戏生态和广告变现。但这不是一个"从0到1创造新品类"的背景，而是"在已知领域做优化和整合"的背景。

与AppLovin的Adam Foroughi（创始人+14年CEO）相比，Bromberg缺乏"从业务本质倒推组织形态"的原生性。他更像是一个被请来"修理坏掉的机器"的人，而非"设计新机器"的人。

**3. AI叙事——GDC 2025的AI工具展示**

Bromberg在GDC 2025展示了AI创作工具（开发者用自然语言生成完整casual game），试图将Unity定位为"AI-native创作平台"。这是正确的方向，但多家公司（包括Roblox、Epic）都在讲类似的AI创作故事。Unity在AI上是否有genuine insight还是在跟随市场叙事，目前无法判断。

**4. 关键减分因素**

- **非创始人turnaround CEO的先天限制**：Bromberg无法像Foroughi（AppLovin）那样以创始人权威推动组织变革。他需要"赢得"团队信任而非"拥有"它
- **前任遗留的信任赤字**：Riccitiello的Runtime Fee丑闻+ironSource合并灾难在开发者社区和内部都造成了深层信任损伤。Bromberg不是制造问题的人，但他承担着修复的责任
- **18个月内高管团队几乎全部更换**：这是turnaround的必要操作，但也意味着新管理团队尚未磨合到位
- **缺乏组织设计方面的公开思考**：目前未见Bromberg对"Unity应该是什么样的组织"有深层思考——他的公开表述聚焦于战略（产品方向、成本控制）而非组织（文化、考核、信息流通）

**D1评分逻辑**：3分对应"中规中矩的管理者"。Bromberg在turnaround执行力上超过3分（果断、聚焦），但在vision深度和组织设计意识上不足3分（无原生组织思考、非创始人权威）。综合给3分，标注turnaround CEO特殊情况。

**信源标注**：公开财报(S5) + 媒体报道(S2) + CEO公开发言(S3)。缺乏内部一手信息。

### D2: Key Leader 配置 [2.5 / 5] ⚠️ 置信度低 — 全新高管班子，ironSource系统性出清，但新团队未经验证

**核心判断：正确的人事清洗+合理的新任命，但团队化学反应尚未被验证。**

**关键人事变动**：

1. **CEO Matthew Bromberg**（2024年5月）：前Zynga COO。如D1分析
2. **CFO Jarrod Yahes**（2025年1月）：前Shutterstock CFO。Shutterstock同样是"创作平台+变现"的双模式公司，背景匹配。但Shutterstock规模远小于Unity，能否驾驭$2B规模尚不确定
3. **CTO Steve Collins**：前King/Candy Crush CTO。King是移动游戏顶级公司，Collins理解"引擎+移动游戏"交叉领域。这是一个精准的CTO任命——King的技术挑战（大规模移动端性能优化）与Unity引擎业务高度相关

**系统性出清的信号**：
- 6位ironSource联合创始人全部离开——这不是个别人事调整，而是整个ironSource势力被清除
- 18个月内高管团队几乎完全更换——与ironSource合并相关的"政治包袱"被一次性清理

**评分逻辑**：
- 正面：人事清洗果断且方向正确；新CTO背景精准；ironSource势力清除消除了组织内部的"两个朝廷"问题
- 负面：新CFO来自小公司（Shutterstock $860M收入 vs Unity $1.85B），scale-up能力未知；除CEO/CFO/CTO外的VP及以下管理层信息几乎空白；全新团队的磨合期风险——通常需要12-18个月才能形成有效协作
- 关键未知：Who is running Vector? 广告平台是Unity当前最重要的增长引擎，但负责Vector的高管是谁、什么背景，完全不知道

**信源标注**：公开任命新闻(S2) + LinkedIn公开信息(S4)。中层管理结构信息空白。

### D5: 组织熵减能力 [3.5 / 5] — 正在大幅熵减，但尚未达到"精简高效"状态

**核心指标：大量bloat被移除，但人效仍远低于行业标杆。**

**关键数据**：

| 指标 | 数值 | 对标 |
|------|------|------|
| 员工数变化 | ~7,800 -> 4,987 (-36%) | 2024年1月裁25%(1800人)，后续持续优化 |
| 收入变化 | FY2024 $1.81B -> FY2025 ~$1.85B (+2%) | 裁36%人，收入基本持平 = bloat清除 |
| 人效 | ~$371K/人 | 远低于AppLovin ~$2M+/人 |
| 重组费用 | $205M + $250M额外节约 | 大规模的一次性出清 |

**熵减力度强，但绝对水平仍低**：

从7,800人砍到4,987人而收入基本持平，说明之前的组织中至少有30%+是纯bloat——这在ironSource合并后的整合失败中完全可以理解。Bromberg的做法（一次性大幅裁员 + 关停非核心业务）是经典turnaround熵减手法。

但$371K/人的人效放在广告tech领域仍然很低——AppLovin ~3,200人支撑~$5B收入（~$1.6M/人），The Trade Desk ~3,000人支撑~$2.4B收入（~$800K/人）。Unity的人效说明要么还有进一步精简空间，要么Create引擎业务的利润率天然低于纯广告平台。

**结构性熵减设计**：
- 关停ironSource Ads Network（2026年4月30日）= 永久性消除一个complexity源
- 探索剥离Supersonic游戏发行 = 进一步聚焦
- 但缺乏"防止未来再膨胀"的结构性机制——Bromberg会不会在增长恢复后重新扩招？

**评分逻辑**：3.5分反映"正在做正确的事（大幅裁员+关停+聚焦），但绝对水平仍不够好（人效远低于标杆），且缺乏防止未来再膨胀的结构性设计"。

### D6: Talent Density [2 / 5] ⚠️ — Runtime Fee + 裁员重创人才储备，恢复中但信号偏弱

**核心判断：2023-2024年的连续冲击（Runtime Fee丑闻、CEO被炒、25%裁员、ironSource整合混乱）几乎必然造成了严重的人才流失，恢复尚需时间。**

**负面信号叠加**：
1. **Runtime Fee事件（2023年9月）**：Unity宣布按安装次数收费，开发者社区爆发，引发内部工程师的深度不信任——"公司在伤害我们服务的用户"。虽然政策被撤回，但对内部士气和外部employer brand的损害是持续性的
2. **CEO被解雇（2023年10月）**：Riccitiello被董事会解雇 -> interim CEO Jim Whitehurst -> 2024年5月Bromberg上任。7个月的领导层真空期
3. **25%裁员（2024年1月）**：1,800人被裁。大规模裁员通常导致"留下来的人也开始找工作"的二次流失
4. **ironSource文化冲突**：ironSource（以色列ad tech, 有adware/malware历史争议）与Unity（游戏引擎, 工程师文化）的文化冲突在合并后从未真正解决，直到ironSource系统被清除

**Glassdoor数据**：
- 整体3.4/5 — 低于tech行业平均（通常3.8-4.2）
- 工程师3.2/5 — **低于公司整体 = 技术公司的黄旗信号**。工程师是Unity的核心生产力，他们的满意度低于非技术岗位说明技术团队的问题更严重
- 职业机会3.1/5 — 裁员后的幸存者普遍感觉晋升通道被压缩
- 文化3.1/5 — 连续变故后的文化认同感薄弱

**微弱的正面信号**：
- 2025年Glassdoor评价开始好转："past their challenging environment"
- CTO Steve Collins（前King CTO）的加入可能对工程团队有正面信号——一个被技术社区尊重的leader
- ironSource势力清除后，文化冲突源被消除

**评分逻辑**：2分反映"连续重创后的低谷期"。工程师评分低于公司整体是硬伤，Runtime Fee事件对employer brand的长期损害不可忽视。如果2026年下半年Glassdoor工程师评分回升至3.5+，可上修至2.5-3。

### D7: Key Bet 战略取舍 [3.5 / 5] — "Burn the Boats" All-in Vector是高风险高回报的正确赌注

**核心判断：Bromberg的战略取舍方向正确且足够果断，但这更多是"纠错"而非"创新"——他在修正Riccitiello时代的错误，而非开辟全新方向。**

**All-in 的核心bet**：

**Bet 1: Vector AI广告平台 — 对标AppLovin AXON**
- Vector占Grow收入56%，Q4 2025达$338M
- 2026年1月YoY+72%（MoM最佳）
- 目标：2026年底Vector ARR超$1B
- 逻辑：用AI重构移动广告bidding，从传统waterfall切换到实时ML优化
- 风险：AppLovin AXON已经在这条路上领先2-3年，且margin高达80%+。Unity是在追赶而非领跑

**Bet 2: 关停ironSource Ads（2026年4月30日）— "Burn the Boats"**
- 这是最大胆的取舍：关停一个仍在产生收入的legacy产品，强迫客户迁移到Vector
- 优点：消除内部资源分散、强制客户迁移、简化技术栈
- 风险：如果Vector无法承接所有迁移流量，部分客户可能转投AppLovin或Google
- 这个决策的"不可逆性"是加分项——表明管理层不是在对冲风险，而是真正committed

**Bet 3: Unity 6 + AI创作工具**
- Unity 6引擎是修复开发者信任的核心载体——Runtime Fee事件后，开发者需要看到"Unity还是那个Unity"
- GDC 2025展示的AI工具（自然语言生成游戏）是正确方向，但竞争激烈（Roblox、Epic都在做类似的事）
- 这个bet更多是"防守"（保住引擎市场份额）而非"进攻"

**Bet 4: 中国市场YoY+50%**
- Unity在中国游戏市场有独特地位（大量中国手游用Unity开发）
- 这是一个地域差异化bet——AppLovin在中国市场渗透较弱

**敢于放弃什么**：
- 放弃ironSource Ads的现有收入（"Burn the Boats"）
- 探索剥离Supersonic游戏发行（非核心业务）
- 放弃了成为"everything company"的野心，聚焦Create + Grow双支柱

**Bet的原创性评估**：
- Vector AI广告 = 跟随AppLovin的方向，非原创
- "Burn the Boats" = 执行层面果断，但策略本身是turnaround标准手法
- AI创作工具 = 行业共识方向，非独特insight
- 总体：方向正确，执行果断，但缺乏genuine original insight

**评分逻辑**：3.5分反映"正确的turnaround战略取舍 + 足够的果断性"，但不到4分因为"这更多是纠错而非创新，Vector在追赶AppLovin而非领跑"。

### A类核心机制识别

**当前Unity缺乏可识别的A类核心组织机制。**

A类机制要求"去掉它公司就垮了"的必要性 + 明确的因果链条 + 战役验证。Unity目前处于turnaround重建期，新的组织机制尚未成型。

**候选B类（支撑机制）**：

**B1: "Burn the Boats"强制迁移机制**
- 通过关停legacy产品（ironSource Ads），强迫内部和外部（客户）同时向新方向迁移
- 这是一种turnaround期特有的组织对齐手段，但非持续性机制——一旦迁移完成，这个机制就不再存在
- 缺乏A类的"持续运转"特征

**B2: Create-to-Grow开发者闭环**
- Unity引擎获取开发者 -> Vector帮开发者变现/获客 -> 数据反馈优化广告模型
- 这是业务闭环，但目前缺乏将其转化为**组织机制**的证据（如何确保Create团队和Grow团队之间信息流通？有没有制度化的协作？）

**与AppLovin的差距**：AppLovin的A类机制是"AXON AI模型飞轮"——200人团队用AI持续优化广告效率，数据规模->模型精度->客户效果->更多数据。Unity的Vector在试图构建类似飞轮，但(1)团队规模大得多（效率低），(2)起步晚2-3年，(3)尚未被margin数据验证。

### 组织风险清单

1. **人效差距是结构性的还是阶段性的？** $371K/人 vs AppLovin ~$1.6M/人。如果Create引擎业务天然需要大量工程师（维护跨平台兼容性、渲染管线等），那么Unity的人效可能永远不会接近纯广告平台。但如果是bloat残留，则还有优化空间。这个判断直接影响Unity的长期margin ceiling

2. **Vector能否在ironSource关停后承接客户？** 2026年4月30日是关键时点。如果大量ironSource客户在迁移过程中转投AppLovin或Google，Unity的Grow收入可能出现断崖。Q1 2026的超指引表现（$505-508M vs $480-490M）是正面信号，但关停尚未发生

3. **非创始人turnaround CEO的"天花板效应"**：Bromberg能把Unity从"灾难"修到"正常"，但能把它从"正常"带到"卓越"吗？历史上，非创始人turnaround CEO成功转型为growth CEO的案例较少。Unity可能需要在turnaround完成后（2027？）重新评估领导力需求

4. **人才流失的二次效应**：连续重创后，Unity最优秀的工程师可能已经流向AppLovin、Epic、Roblox等竞争对手。新CTO Steve Collins能否重建工程团队的talent density是2026-2027年的关键观察点

5. **"追赶者"战略的固有风险**：Vector追赶AppLovin AXON，Unity 6追赶Unreal 5/6，AI工具追赶行业共识。当一家公司在所有核心赛道上都是追赶者时，资源分配的优先级错误的风险极高

### 与知识库案例的对标

**最接近的对标：Meta 2022-2023 turnaround（Year of Efficiency）**

| 维度 | Meta (小扎 Year of Efficiency) | Unity (Bromberg turnaround) |
|------|------|------|
| 问题根源 | Metaverse overinvestment + 膨胀至87,000人 | ironSource合并灾难 + Runtime Fee丑闻 + CEO被炒 |
| CEO身份 | 创始人，绝对权威，亲自主导 | 非创始人职业经理人，需要赢得信任 |
| 裁员力度 | ~21,000人(-24%) | ~2,800人(-36%) |
| 人效变化 | 显著提升（$1.5M -> $2M+/人） | 改善但仍低（$324K -> $371K/人） |
| 核心bet | AI + Reels（已验证，广告收入重回增长） | Vector AI广告（增长中但未达标杆margin） |
| 文化重建 | "Year of Efficiency"成为新文化叙事 | 无明确的新文化叙事 |
| 结果 | 成功：股价从$90回到$600+ | 进行中：股价在$20-30区间（远低于IPO $50+） |

**核心差异**：Meta的turnaround由创始人CEO亲自主导，拥有绝对权威和文化影响力。Unity的turnaround由外聘职业经理人执行，缺乏这种权威。Meta在turnaround完成后迅速进入AI增长期，Unity的增长引擎（Vector）仍在追赶阶段。

**这个差异是决定性的**：创始人turnaround vs 非创始人turnaround的成功率差异巨大。小扎可以用一句"Year of Efficiency"重塑整个公司文化，Bromberg做不到这一点。

**与AppLovin的对标（直接竞争对手）**：

| 维度 | AppLovin (Foroughi) | Unity (Bromberg) |
|------|------|------|
| CEO类型 | 创始人，14年坚持 | 外聘turnaround CEO |
| 员工数 | ~3,200 | ~4,987 |
| 收入 | ~$5B | ~$1.85B |
| 人效 | ~$1.6M/人 | ~$371K/人 |
| AI广告 | AXON领先2-3年，80%+ margin | Vector追赶中，margin未公开 |
| 组织特征 | 精英小团队，高度聚焦 | 正在精简，但仍臃肿 |

**人效差距4.3倍是最刺眼的数字。** 即使考虑Unity有Create引擎业务（需要更多工程师），这个差距也暗示Unity的组织效率仍有巨大改善空间。

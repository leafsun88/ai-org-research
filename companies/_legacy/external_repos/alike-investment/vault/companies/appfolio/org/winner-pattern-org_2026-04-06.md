# AppFolio — D1-D7 组织穿透

## Step 0: 业务本质画像

**业务本质**: 物业管理垂直SaaS（单一行业、深度功能）
**理想组织形态**: 产品驱动 + 客户成功 + AI赋能 + 高效运营
**关键能力要求**: 产品研发 + 行业理解 + AI应用 + SMB销售+服务

---

## D1: CEO认知质量 — 2.9/5

**依据**:
- Shane Trigg有25年SaaS经验（Salesforce/MRI Software）
- "AI-native Performance Platform"叙事有方向感
- 推动效率化改革（op margin提升）

**不足**:
- 非创始人CEO → 使命感稀释
- 缺乏独特洞察（Salesforce标准SaaS打法）
- 任期仅3年，未经历完整周期验证
- 裁员方式引发文化危机 = 执行粗糙

## D2: Key Leader Pipeline — 2.5/5

**依据**:
- 创始人Klaus Schauser仍在Board → 有监督
- 信息不足：除CEO外的核心高管信息有限

**不足**:
- 创始人退出日常运营 → 知识传承风险
- 经历多轮裁员 → 人才流失风险
- 没有明显的"下一个CEO"候选人

## D3: 组织架构适配 — 3.0/5

**依据**:
- 垂直SaaS不需要复杂组织
- 单一行业聚焦 → 组织对齐相对容易
- AI嵌入产品 → 技术团队重要性提升

**不足**:
- 裁员后组织可能过度精简
- 多轮裁员暗示此前组织臃肿

## D4: 文化-战略一致性 — 2.5/5

**依据**:
- "AI-native"战略与产品方向一致
- 98%客户使用AI → 产品层面执行到位

**不足**:
- Glassdoor差评: "CEO烧毁文化"、"裁员残忍"、"有毒环境"
- 从"人本文化" → "纯利润驱动" = 文化断裂
- 薪酬竞争力差（3.4/5）→ 人才留存风险
- 文化-战略严重不一致: 要做AI-native但不善待做AI的人

## D5: 组织熵减 — 3.0/5

**依据**:
- 裁员本身是熵减动作
- Op margin从低位提升至25%+ → 效率改善
- 单一垂直 → 天然低熵

**不足**:
- 裁员方式产生了新的组织熵（恐惧文化、信任缺失）
- "有毒环境"反馈 = 内部摩擦增大

## D6: Talent Density — 2.5/5

**依据**:
- 工程师WLB/D&I评分高（4.4/4.6）
- Santa Barbara总部 → 吸引力有限

**不足**:
- 薪酬3.4/5 → 不具竞争力
- "偏袒、恐惧和操控"的描述 = talent density可能在下降
- 优秀人才可能在裁员后流失

## D7: Key Bet质量 — 3.0/5

**依据**:
- AI-native平台化 = 正确的方向
- Premium Tier = ARPU提升策略
- 98%AI渗透率 = 执行到位

**不足**:
- AI在物业管理中的应用深度有限（不是paradigm shift）
- 没有"一个大赌注"定义公司未来
- 物业管理市场的TAM天花板明确

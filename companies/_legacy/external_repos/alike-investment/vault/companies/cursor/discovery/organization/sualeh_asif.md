## Product Vision & Technical Philosophy

### Core Product Beliefs (from public interviews)
- Cursor aims to eliminate "low entropy actions" -- predictable patterns and repetitive tasks in coding
- The goal is a harmonious hybrid of human and AI capabilities, with AI as a powerful assistant
- Tools should "increase productivity and learning, not replace the coder"
- Focus on speed and user experience as differentiators

### Key Technical Contributions (as CPO)

**Tab Feature**
- Custom AI models predict and apply entire code edits (beyond single-line autocomplete)
- Extends to multi-line edits, jumping to different locations in the same file
- Can suggest terminal commands
- Uses speculative edits and multi-query attention for fast response times

**Codebase Indexing** (announced on X, April 2025)
- Supports 1M+ files / 1B+ lines of code
- Warm indexes: initialization 12-20x faster with near-instant search speed
- Cold initialization: 5x faster
- Custom retrieval systems using Merkle trees to avoid storing source code on server

**Infrastructure Scale**
- ~100M model calls per day
- Multi-cloud inference infrastructure
- Orchestrator service "Anyrun" built in Rust, uses AWS Firecracker for process isolation
- Migrated from Yugabyte to object storage (S3/R2/Blob) for large tables

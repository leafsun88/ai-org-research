# Midjourney V7 Showed Up in Crocs and COOKED — Source Note

## V7 让老用户重新回到 drawing board，模型升级会重置一整套社区知识

这期的核心价值在于它不是看几张图下结论，而是两个重度用户在重新测试 V7 的“驾驶方式”。他们回到旧 prompt、旧参数、旧内容资产里，重新判断 stylize、personalization、remix、draft mode、explore page 和图像质量的边界。Midjourney 的产品速度因此不是单纯模型发布速度，还包括社区能否在每次大版本后快速把隐性参数知识重新建起来。V7 早期争议很大程度上来自用户把旧模型习惯直接套进新架构，结果误判了模型能力。

“新模型出来时，我喜欢回去翻旧 prompt，看看它们现在还能不能工作；我会看以前那些没分享过的东西，想起当时为了某个画面做过什么，然后也会意识到，过去觉得革命性的图现在看起来已经像旧资产了。真正有用的是重新测试参数和 feature，因为 Midjourney 里很多 incremental changes 不会明说，你只有回到 drawing board 重新实验，才会发现这些大版本和模型变化到底改了什么。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-13

## Stylize 的甜点下移，说明 V7 默认审美更强但旧参数习惯不能照搬

两位主持人反复讨论 stylize 从过去常用的 700、800 回到 150、200、300 这样的区间，这个经验不是简单教程，而是模型默认审美权重变化的信号。V7 已经能在较低 stylize 下给出稳定、漂亮、有控制感的图像，用户不需要再靠极高参数强行拉效果。对专业用户而言，这意味着所有历史 prompt、模板和参数组合都要重新调校；对 Midjourney 而言，每次模型升级都在改变用户的手感，教育和社区复盘会变成产品的一部分。

“以前我会把 stylize 推到很高，上一代模型里我经常跑到七八百，只是为了把某些东西往前推；但现在我发现最好的结果反而在 150、200、300 左右。V7 自己已经有更多审美重量，不需要把每个旋钮都拧到头；旧的参数习惯不会完美迁移，每一次大模型更新都像教你一种新的驾驶方式。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-13

## Remix、style reference 和 mood board 让用户把“构图”和“风格”分层处理

Rory 对 remix 的讲法很像真实创意 workflow：先控制基础 image construction，再用 remix strong、style reference、style code 或 mood board 往上叠风格。这个机制说明 Midjourney 的高级用户不是一次 prompt 祈祷结果，而是在把构图、角色、质感、风格和变体拆成多个操作层。Midjourney 相比简单图像工具的优势，也在这种层级化控制里：它不是只有文本到图像，而是允许用户在生成以后继续把输出当材料反复进入系统。

“我喜欢 remix 的原因是，有时候 style reference、style code 或 mood board 会影响整张图的 construction，甚至把画面整个拧走；我想先控制 prompt 本身和输出结构，得到一个基础画面以后，再用 remix 加 style reference、style code 或 mood board，让它叠在上面，而不是一开始就把整张图重塑。很多时候我甚至不改任何字，直接 remix strong 再跑一次，它像介于 very subtle 和 very strong 之间的另一种 variation。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-13

## Character reference 和 omni reference 的真正价值，是把 Midjourney 拉向产品、角色和商业资产

这期把 character consistency 和 omni reference 放在一起讨论，重点不只是“同一个人像不像”，还包括 object reference 对产品图、品牌资产和客户工作的影响。Midjourney 过去最强的是审美和 surprise，弱点是重复控制和真实商业资产的可交付性；如果 omni reference 能同时解决人物、物体和产品一致性，Midjourney 会更直接进入 freelancer、agency、marketer 的工作流。这里也能看到用户对公司定位的拉扯：Midjourney 似乎不太想被理解成 corporate tool，但大量核心用户已经在把它用于客户项目。

“大家都在说 character consistency，但不要忽略 omni reference 里的 object reference；如果它真的好用，那会打开完全不同的思考方式。很多人想把自己的产品放进去，这也是它和 ChatGPT 竞争的地方。很奇怪的是，Midjourney 好像不太想让人把它用于 business，但我们这些人就是想这么用；现实是有很多 creative people、freelancer、scrappy marketer，不是在一万五千人的大 agency 里，而是在 sweatpants 里给客户做很酷的东西。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-13

## V7 更像可定制 base model，personalization 正在变成默认入口

Rory 把 V7 比作旧 Stable Diffusion 那种需要在 base model 上定制的架构，这个比喻很有判断力：V7 的好结果越来越依赖 personalization，用户要通过 ranking 把自己的审美、构图偏好、颜色和质感喂给系统。他提到前 200 张 ranking 后，手、appendage、颜色和 composition 立刻变好。这个机制对 Midjourney 的 retention 很强，用户投入的偏好数据越多，越难迁移到其他工具；但它也增加了入门摩擦，casual 用户可能不愿意先“训练”工具。

“我的观察是，V7 的 base model 有点像更早那种 Stable Diffusion base model，是一个要在上面 build off、customize off 的基础；Midjourney 这次明显把 personalization 做成 standard。你进去 rank 一些图，前 200 张之后，我的图立刻变好，手、appendage、颜色、composition 都改善了，而且我 rank 得越多，结果越像我想要的东西；所以如果你还没给 V7 做 personalization，先给它 200 张再判断。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-13

## Draft mode 和 voice 改变 prompt 节奏，Midjourney 开始靠更短 feedback loop 支撑探索

Draft mode 在这期里被视为一种全新的 prompting 方法，而不是低质量版本。它让用户快速排除方向、用语音在手机上继续引导，缩短从想法到视觉反馈的周期。Midjourney 如果要从图像生成器进入专业创作工具，探索速度和最终画质同样重要：draft mode 负责广泛试方向，高质量生成负责交付。voice 的加入尤其重要，因为它把移动端从难以输入长 prompt 的地方，变成可以边走边描述、边想边改的创作界面。

“Draft mode 已经比刚发布时好一些了，它是一种完全不同的 prompting 方式；你可以用 voice 来引导它，这很有意思，因为你不再需要在手机上用两个手指打一整段 prompt，也不用担心参数少了空格就全坏掉。它更像是在你还没完全知道自己要什么时，先快速试方向、排除错误路线，再继续往下走；这感觉像一个自然的下一步。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-13

## Explore page 变好以后，社区作品本身成了新的 prompt 和风格发现界面

两位主持人明显感受到 V7 后 explore page 的基线质量提升，开始把它当作 Pinterest、Instagram save 或 prompt research 的组合。这个细节说明 Midjourney 的社区界面不仅是展示结果，也是学习系统：用户通过别人用的 style references、prompt 长短、构图、颜色和参数，反推当前模型什么有效。Midjourney 如果能把 liked images、style refs、folders 和 search 更好组织起来，社区内容会变成留存和再创作的核心资产，而不是一次性 gallery。

“Explore page 这几天明显好多了，baseline quality across the board 非常明显，我现在会真的从里面拉 prompt 出来，这以前几乎不会做。它很像 Pinterest pinboard 或 Instagram save，我会存一些以后可能用到的东西，但现在的问题是 liked images 只能线性往回滚，很多非常好的 inspiration 会死在那里；同时看别人怎么 prompt 也很有趣，因为没有一种固定结构，你能从 hot page 上感受到 V7 里什么 architecture 正在工作。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-13

## V7 的亮度、动态范围和纵深改善，是 Midjourney 审美护城河的微观证据

这期后半段对 V7 图像亮度、动态范围、纵深和细节的讨论很细，价值高于一般“好看”判断。主持人把 Midjourney 和 Google/Runway/Imagen 等结果对比，注意到 V7 不再像旧模型那样偏暗或用大面积黑暗掩盖细节，而是在更亮画面里保住 cinematic color grade、foreground/background detail 和脸部光影。这类微观视觉改进是 Midjourney 护城河的一部分：它不是单一 prompt adherence，而是长期训练审美、细节和 creative interpretation 的模型品味。

“我看那个对比视频时，第一眼注意到的不是原帖重点，而是 Midjourney 明显亮了很多；它在保持 cinematic color grade 的同时让画面更亮，这其实很难，因为更亮意味着每个地方都要有更多细节，不能靠黑色区域糊过去。以前很多模型像是默认欠曝，黑的地方不用生成太多东西；V7 现在能在亮部、暗部和中间调里保留更多 dynamic range，背景草地、人物头发、脸上的多方向光都更真实。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-13

## Midjourney 的核心差异仍是 creativity/coherence 平衡，而不是单纯更听话

主持人最后把 Midjourney 和其他图像工具的差异压到一个关键点：许多工具都在追求 coherence，Midjourney 则一直保留 creativity 和微观 surprise。stylize、weird token、极端 prompt、tonal contrast、micro detail 这些讨论，看似是发烧友参数聊天，实则说明 Midjourney 用户愿意为不可完全预测的审美探索付出学习成本。对公司而言，这既是护城河也是风险：专业用户爱它的“有性格”，普通用户可能更想要 assistant 式听话。

“很多其他 image generator 都更专注于 coherence，而 Midjourney 一直更专注 creativity；我想做这个概念，但让它更荒谬一点，或者把一个 token 往更 gnarly 的方向推，这种能力很少见。它的关键是 coherence 和 creative interpretation 的平衡，因为我仍然喜欢那些 categorical tokens 带来的意外结果；只要你给它参数，创造性就会产出一些真正的 bangers。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-13

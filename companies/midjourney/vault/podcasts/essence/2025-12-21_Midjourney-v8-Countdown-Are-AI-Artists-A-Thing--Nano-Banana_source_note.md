# Midjourney v8 Countdown, Are AI Artists A Thing? Nano Banana — Source Note

## “AI artist” 争议背后，商业创作者把 Midjourney 当生产工具而不是身份标签

节目开头花了大量时间讨论 AI artist 这个词，真正有用的点不是站队，而是区分商业资产生产和个人艺术表达。两位主持人都不太愿意在客户工作里自称 AI artist：他们更像是在用 Midjourney、Nano Banana、Weavy 等工具解决 marketing asset、视觉方案、workflow scale 和客户交付问题。这对 Midjourney 的用户分层很重要：一部分社区会围绕 art 身份和审美表达展开，另一部分高频付费用户其实在乎的是更快、更稳定、更可复制地完成工作。

“如果是 work related 的东西，我不会说自己是 AI artist；我觉得 AI artist 这个词太 polarizing，也不一定是好描述。很多时候我们是在给客户做 marketing assets，目标是找到最好的路径到达结果，而不是把一张图挂到墙上。就像我以前做 graphic designer，也不觉得自己在做 art，只是需要生产资产，把它们做得好看、能用、能完成任务。” —— Drew / Rory, Midjourney Fast Hours, 2025-12-21

## 单张生成图不稀缺，稀缺的是 theme、story 和持续审美系统

两位主持人都认为单纯按按钮生成一张图很难自动成为 art，真正的创作者价值在于把图像组织成更大的 theme、series、narrative 或多模态作品。这个判断对 Midjourney 很重要：模型越强，单图越泛滥，用户的价值就越从“会不会 prompt”转向“有没有可识别的 vision”。Midjourney 如果想保住专业创作者心智，就要帮助他们建立连续风格、作品集、故事线和可复用视觉系统，而不是只提供一次性漂亮图。

“如果我只是把这些图发出去，它们还不是 art；我手选了几张图、调了 mood board，这只是故事的一部分。问题是 theme 是什么、story 是什么、我用什么 format 讲它，是一堆图片、一个 multimodal piece，还是一百张图背后的世界观？现在大家都有成千上万甚至几十万张图可以分享，真正难的是怎样把它们 cohesively tell something，而不是再抽到一张好看的图。” —— Drew / Rory, Midjourney Fast Hours, 2025-12-21

## Mood board 和 style creator 把 Midjourney 从出图工具推向个人视觉语言产品

这期提到 style creator 已在 alpha.midjourney.com 软启动，配合 mood board 使用，很像 Midjourney 在把“个人风格”产品化。用户不再只是输入 prompt，而是在训练、选择、滚动、收藏、复用一套 aesthetic profile。Style creator 的连续滚动选择，比旧的 grid-by-grid 更顺滑，也降低了创建 style 的摩擦。这个方向和 V7 personalization、sref、mood board 是同一条线：Midjourney 想让用户回到自己的风格资产，而不是每次从零开始。

“Style creator 已经在 alpha site 上 live 了，不是在普通 midjourney.com 上；他们像是在 alpha.midjourney.com 先 soft launch 新更新。现在体验也改了，不再是一格一格选完点 next，而是连续 scrolling，你喜欢就点，右边的 style 会实时更新；这样不用一直 click，感觉没那么 cumbersome，也更像是在一点点把自己的风格调出来。” —— Drew / Rory, Midjourney Fast Hours, 2025-12-21

## V8 推迟让 Midjourney 更需要小更新 cadence 来维持社区信号

节目里没有拿到 V8 明确时间，主持人判断可能推到明年。这个空窗期的关键不在“delay”本身，而在 Midjourney 能否延续 V7 后的小更新节奏：style creator、editor model、V8、edit model、video、Q 系列质量层，都需要以可感知的方式持续出现。社区对 Midjourney 的耐心来自“它会做出好东西”，但社媒热度和专业用户注意力越来越分散；如果大版本继续拖，小功能和 alpha release 就承担了保持信任的角色。

“我没听到什么新的 V8 消息，感觉像是推到明年了；这次我去度假，竟然第一次什么都没发生，office hour notes 里也几乎都是 style creator。希望它能像 V7 后那样，回到小更新持续掉落的节奏，因为那段时间真的很棒；如果只是一直等一个大版本，社区很难一直保持同样的注意力。” —— Drew / Rory, Midjourney Fast Hours, 2025-12-21

## AI 工具发布太快，专业用户的评测窗口从几小时压缩到几分钟

主持人讲自己现在测试新模型时，不再像以前花几小时，而是用少数固定 protocol 跑几分钟，过不了就先放下。这是 AI 工具市场的重要现实：模型和功能发布太密，创作者无法深度试完所有东西，只会把时间给那些第一印象强、质量稳定、工作流明确的工具。Midjourney 的优势是每次大 release 往往能让重度用户愿意深入，但它也必须面对用户 attention budget 变短的事实；发布太早、承诺太多、第一轮体验不好，可能直接失去下一次测试机会。

“现在不是 pace of development 快这么简单，而是我以前会花几个小时测试一个东西，现在只给它几分钟；我有几个固定的测试 protocol，图像模型或视频模型如果一发布连那些都做不好，我就觉得先没用。现在选择太多了，Luma、Kling、Google、Runway、Midjourney、ChatGPT，每个都在发东西，用户没有时间深挖所有工具，所以 release 一定要在第一时间让人觉得 noticeably better。” —— Drew / Rory, Midjourney Fast Hours, 2025-12-21

## Video 和 world model 会把 Midjourney 推向更重的技术战场

节目中从视频模型聊到 world model，说明 2026 年创意 AI 的竞争正在从图像质量进入物理一致性、长序列、声音、角色连续性、空间理解和 3D/世界模拟。Midjourney 的图像审美优势能不能迁移到视频，并不是自动成立；视频需要更高算力、更强一致性和更复杂编辑控制。对 Midjourney 来说，V8 与后续 video 不只是产品功能，而是小团队能否继续在更重、更贵、更跨模态的战场保持 taste 和执行力的考验。

“我看到有人说，everyone will have access to their own world simulator，视频模型训练到足够规模就会变成 world model，因为要预测下一帧就必须学习世界怎样运作。2026 年我们会进入这个讨论，视频里不仅是画面，还有声音、lip sync、physics、3D、甚至 touch、smell、taste 这些更远的 modality；这会比图像难很多，Midjourney 的审美优势能不能带过去，是下一阶段的大问题。” —— Drew / Rory, Midjourney Fast Hours, 2025-12-21

## ChatGPT 和 Nano Banana 抢的是入口，Midjourney 仍被专业用户拿来对标最高审美

主持人对 ChatGPT image update 和 Nano Banana 的比较很直：ChatGPT 胜在已经坐在每个人电脑上，但图像仍有偏黄、airbrush face、真实感不足的问题；Nano Banana 在某些真实感和细节上更强，甚至可以做出接近 Midjourney 的审美。这个竞争格局说明 Midjourney 的大众风险来自入口和便利，而非所有竞品都在审美上超过它。专业用户会继续比较 realistic aesthetic、风格控制和可交付性；普通用户可能选择“已经在 ChatGPT 里、够好、方便”的工具。

“ChatGPT 的 image model 还是有那种 3300 Kelvin 的暖黄调和 airbrush face，两个我最希望它修的问题都没修；Nano Banana 的一些例子明显更自然，左边像现实，右边像加了滤镜。问题是 ChatGPT 已经在几乎每个人电脑上，这是它的 surface advantage；但如果只看真实感和审美，很多例子仍然不接近，专业用户还是会把 Midjourney 或 Nano 这类结果拿来当更高标准。” —— Drew / Rory, Midjourney Fast Hours, 2025-12-21

## Node workflow 把创作从单图 prompt 推向 shot list、contact sheet 和可复用系统

Rory 展示的 Weavy/Nano Banana 工作流是这期最强的实操证据：先用视觉参考和 LLM 描述建立 aesthetic guide，再生成一个 unbranded studio car 或产品基础图，套上 Kodak/Polaroid 等包装风格，做 front/rear/low-angle/high-angle shot list，再用 list selector 和 keyframe 串成视频。这个过程说明专业 AI 创作已经不是“写一句 prompt”，而是把 reference、系统 prompt、shot types、composition rules 和批量生成组织成可复用 pipeline。Midjourney 如果缺少这种 workspace，用户会把它作为起点，然后去 Weavy、Flora、Kling 等工具完成系统化。

“这个 workflow 的关键不是一句 prompt，而是先找到视觉参考，让 LLM 描述 aesthetic，再把它用到车上；我会先让它生成一个 unbranded studio car，因为背景越少，processing power 越集中在车上，然后把 Kodak packaging 的 linear geometry、synthwave gradients、CMYK spectrum、rainbow、typography 和 branding elements 应用上去。接着我做 shot list：保持同一辆车、同一环境，换 front view、rear low angle、high angle three-quarter view，再用 list selectors 分到不同节点里，像把所有角度摆在桌上一样决定怎么讲故事。” —— Drew / Rory, Midjourney Fast Hours, 2025-12-21

## 客户越来越喜欢 imperfect、low-fi、candid，AI 工作流要学会“破坏完美”

这期后半段讲 car/video workflow 时，主持人强调客户正在喜欢 imperfect、low-fi、candid、lived-in、not staged 的画面。这是重要审美信号：AI 图像最容易陷入过度干净、过度中心构图、过度广告片质感，真正商业可用的素材反而要主动加入 red-eye、motion blur、overexposed、crop off forehead、negative space 等“错误”。Midjourney 的竞争力如果只是完美图像，可能很快被同质化；如果它能帮助用户精确控制不完美，就更接近真实创意制作。

“很多客户现在真的喜欢 imperfection、low-fi、candid、lived in、not staged、amateurism 这些东西；这就是你要知道规则，然后知道怎么打破规则。我们会在 system prompt 里写 subjects must never be centered、用 rule of thirds、left framing、negative space、cut off foreheads、reject standard portrait framing，也会要求 disposable camera style、red eye、motion blur、overexposed、light leak。真正讲故事的不是完美，而是这些看起来真实、有生活痕迹的破损。” —— Drew / Rory, Midjourney Fast Hours, 2025-12-21

## Negative prompting 会从修补项变成创意系统的一半

节目结尾提到一个很有预见性的 workflow 变化：未来 negative prompting 可能比 positive prompting 更重要。过去用户习惯“说你想要什么”，但多模态 LLM + image generator 组合开始能理解规则、约束和反例，系统 prompt 里“不要什么”会变成塑造风格、减少俗套和稳定批量输出的关键。这个判断对 Midjourney 和同类工具都重要：当模型更听话、更强时，创意控制会从命令图像对象，转向定义边界、禁区、质感和不可接受的默认套路。

“我对 2026 的一个预测是，negative prompting 会比 positive prompting 更重要；我的 system prompts 现在已经有更多篇幅在写我不想要什么，而不是我想要什么。过去我们习惯进入 image 和 video 工具时只说自己要什么，但像 Nano 这种 LLM 和 image generator 结合的东西，让你不用担心说了一个词就被误解成想要它；你可以写规则和 guidelines，写我不要这个、这个、这个，也要这个、这个、这个，创作体验会因此改变。” —— Drew / Rory, Midjourney Fast Hours, 2025-12-21

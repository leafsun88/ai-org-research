# Midjourney's Secret Playground — Source Note

## Drip release 把 Midjourney 社区重新点燃，发布节奏本身变成产品机制

这期从开头就把 V7 后的 release cadence 讲得很清楚：连续掉落 weird、editor 等小功能，比继续等待一个巨型版本更能维持社区能量。Midjourney 的社区不是被动接收产品公告，而是每次新参数、新编辑能力、新模型 patch 都会立刻测试、发图、写教程、争论。对小团队来说，drip release 等于把社区变成外部 QA、教育和传播网络；它也降低了一次性大版本被 ChatGPT 等竞品抢走叙事的风险。

“现在这种 dripping system 比继续等 V7 好太多了，我宁愿他们就这样放出来，让我们玩，然后继续下一个东西；它能把 hype cycle 保持住。五六周前，Twitter 上 Midjourney 的东西好像有点安静，现在因为不断有新功能，所有人又回来了，大家都在测试、玩、发结果。它不是一个巨大的 release 把所有东西都淹没，而是每次都有新的东西让人重新进来。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-22

## Weird 和 chaos 是探索 latent space 的旋钮，不是稳定交付的控制面板

主持人对 weird 和 chaos 的使用边界讲得很成熟：它们不是为了拿到可控结果，而是为了离开默认审美，找到用户自己 prompt 不出来的样式、构图或情绪。高 weird、高 chaos 的价值在于“挖矿”而不是交付，先得到一个意外结果，再把它变成 style reference、mood board 或下一轮 prompt 的起点。这个机制体现 Midjourney 和纯 assistant 工具的差异：它保留了一套故意让用户迷路的探索旋钮，服务的是视觉 discovery。

“Weird 会给你一些你平时根本拿不到的东西，它不是控制工具，而是让我去探索 Midjourney landscape；我经常把它当成 mining style references，用 personalization 关掉的状态去找一些自己不会 prompt 出来的东西。高 weird 和高 chaos 是 Midjourney 产出最独特结果的地方，你不是为了得到最终图，而是为了找到一个可以拿回去当 style reference 或 mood board 的方向。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-22

## Personalization 会把探索拉回个人 profile，Midjourney 的惊喜和粘性在同一机制里

Rory 提醒 weird 开着 personalization 时会被 profile code 拉回个人审美，关掉以后 variety 才真正打开。这个细节很适合分析 Midjourney 的双重产品逻辑：personalization 增加粘性和效率，让用户更快得到“像自己”的图；但当用户想探索陌生风格时，个性化反而会收窄可能性。Midjourney 的高级 workflow 因此会在 personal taste 和 raw exploration 之间来回切换，这比单纯“模型更懂我”更复杂，也更像专业创作工具。

“如果 weird 和你的 profile 一起开着，它还是会很贴近你的 profile；我一般会把 personalization 关掉，因为这样 variety 会大得多。对我来说，weird 更像在 Midjourney 的脑子里玩，看看它能去哪里，找一些我 prompt 不出来的东西；如果我想要控制，那就不是这些参数的用途。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-22

## 新 editor 把生成结果接到后期制作，但当前价值先体现在 smart select 和产品资产实验

这期对 editor 的讨论很具体：retexture、resize、paint、smart select、layers 终于被放到同一个地方，用户开始用它做鞋子、背景、人物脚部、阴影和风格重塑。它还不是完整 Photoshop 替代，但已经把 Midjourney 从“生成一张图”推进到“在已有图上继续制作资产”。尤其 smart select 对 object replacement 和 product shot 的价值很明显：用户可以从一双鞋出发，把它放进不同人物、街景、风格和光影里，测试商业素材的可能性。

“Editor 现在把 retexture、resize、paint、smart select、layers 放在一起，这本身就很好；我用 smart selection 把鞋子从一张图里拿出来，不再只是产品照，而是加上脚、换不同人、换不同环境。它甚至会把鞋子的颜色和风格扩展到整张图里，自动匹配阴影、光线和街面质感；如果只用 Photoshop 做，从这双鞋到这些 lifestyle images 会非常疯狂，但在这里它已经能作为一个实验流程跑起来。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-22

## 编辑器暴露了模型架构迁移成本，V7 不是所有旧功能的即时升级

主持人注意到同一个 editor task 上 V6.1 往往比 V7 更稳，尤其在把产品塞进人物或场景时，V7 图像质量高但结构和组合更容易坏。这是一个很好的产品证据：Midjourney 的 feature set 并不是跟着 base model 一起无缝升级，retexture、mood board、editor 等能力需要围绕新模型重新适配。每次底层架构变化，都会带来功能迁移债务；这也是 Midjourney 需要 drip release 和用户耐心的原因。

“我试同一张鞋图、同一个 prompt，只是换 V6.1 和 V7，V6.1 在 editor 里明显更 coherent；V7 的质量看起来更像 V7，但结构会把腿和组合弄坏。你能感觉这个 feature 原本是为 6.1 engine 做的，到了 V7 有点断开，所以有些东西现在还需要时间重新接上；这就像 mood board、retexture 一样，不是新模型一来所有旧功能都自动变好。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-22

## Q4、super sref 和 creative mode 显示 Midjourney 正在为专业用户加更细的质量层

Office hour notes 里提到 Q4、更强的 super sref、prompt accuracy、anatomy、artifact reduction 和可能的 creative mode，这些信号都指向同一件事：Midjourney 正在把专业用户最在意的质量层和控制层继续细分。Q4 代表愿意用更慢 render 换更高质量；super sref 强化其最有差异化的风格控制；creative mode 则试图在不堆满 slider 的情况下给用户更多可控创意空间。这些不是大众娱乐功能，而是重度创作者会愿意付费和等待的工作流能力。

“Office hours 里提到他们在实验 Q4，也就是用更慢 render 换更高质量；这太适合我们了，因为我宁愿等一天拿最高质量，也不想立刻拿一张只有一半好的图。他们还提到 super sref、prompt accuracy、anatomy、artifact reduction、甚至一个新的 creative mode；sref 对我来说仍然是 Midjourney 最强的 differentiator，比大多数工具的 style reference 都好太多。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-22

## Video 延迟不是单纯慢，而是 Midjourney 品牌必须守住 extraordinary 阈值

节目里提到 video model training 预计月底完成，调校和发布可能还要一到两个月，主持人宁愿等待 extraordinary 版本。这体现 Midjourney 的产品约束：它已经被用户和社区赋予“高审美、高 surprise”的品牌预期，视频如果只是功能清单式追赶，很容易稀释品牌。风险是 Runway、Kling、Google、OpenAI 等竞品继续推进；优势是 Midjourney 可以用更少但更惊艳的 release 维持 premium creative engine 的定位。

“Video model 训练预计月底完成，但 tuning 和 release 可能还要一两个月；如果这意味着最后出来的是 extraordinary 的东西，我宁愿等。V7 已经重新有 hype 了，他们不需要为了说自己 ship 了 video 而毁掉一件好事；这次 V7 的 blowback 很多是因为大家等太久，又正好撞上 ChatGPT image update，然后很多人只跑几个 prompt 就下结论。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-22

## ChatGPT、Photoshop 和 Reeve 的对比说明，Midjourney 的威胁来自入口和肌肉记忆

这期有一段很有用的竞争判断：Photoshop 不会因为 AI 修图出现就消失，因为它有多年积累的功能和用户肌肉记忆；ChatGPT image generation 也会吃掉一部分原本可能属于 Midjourney 的使用场景，因为它已经在用户的日常入口里。Reeve 质量不错但缺乏持续使用 surface。Midjourney 的护城河不是所有功能都第一，而是重度用户已经形成在 Midjourney 创作、调参、找风格、再交给其他工具的习惯。未来竞争会围绕“谁能成为三个核心工具之一”展开。

“Photoshop 不是因为某个 AI 功能就死了，它仍然能做很多别的工具做不了的事；我想找十个理由不付 Adobe 钱，但它还是在 dock 里。ChatGPT 的 image gen 确实改变了我有些时候会不会用 Midjourney，因为它就在每天使用的入口里；Reeve 也很好，但它只是另一个我要去的工具。大家现在不是要二十个工具，而是要三个工具，谁能把这些能力放在一个已经被采用的 surface 里，谁就更有机会。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-22

## Flora、Weavy 和 Patchwork 暗示 Midjourney 下一步要解决“把想法串起来”的 workspace 问题

主持人展示 Flora、Weavy 这类 node-based workflow，说明专业创作者越来越需要把图像、视频、LLM、reference、shot list、keyframe 和变体放在同一张工作台上。Midjourney 当前强在单张图和风格探索，但当用户要做 storyboard、产品系列、视频 keyframe 或多角色一致性时，切换 tab、滚动找图、手动搬运 reference 会打断思路。Patchwork 如果能和生成、编辑、视频打通，Midjourney 可能从 image generator 变成真正的 creative workspace。

“我喜欢 Flora 或 Weavy 这种 node-based workspace，因为你可以把图片、prompt、Gemini、视频、不同角度和 shot list 放在一起，不用在不同地方来回找；它像把想法串起来，而不是滚半天找上一张图。Midjourney 已经有 Patchwork 的雏形，如果能把某种 patchwork canvas 接到生成、编辑和视频里，让 stills 到 animation 不用切 tab，这会是很大的 unlock，因为所有工具和你要完成的东西都在同一个地方。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-22

## 功能变多以后，Midjourney 需要主动教育用户，否则复杂度会反噬 adoption

这期最后很明确地说，editor、layers、weird、remix、style reference、mood board、draft、Q4、creative mode 等功能越来越多，Midjourney 不能只把它们丢给用户自行摸索。早期 power user 愿意挖，但普通创作者和商业用户需要官方示范、教程和清晰 workflow。Midjourney 的小团队过去靠社区教育放大 adoption，到了 V7 以后，官方也要更认真地“drink their own champagne”，否则复杂功能会被低估或闲置。

“像 editor 里 layers 这种东西，Midjourney 需要继续教育大家怎么用每个小功能；不然他们花了这么多时间 build in，最后没人用，也承受不起。哪怕只是 micro feature，像 weird 回来了、remix 回来了，都能让一大群人有东西谈、有东西试，但如果功能越来越多，他们就必须更好地 drink their own champagne，直接展示自己到底怎么用、怎么把这些东西串起来。” —— Drew / Rory, Midjourney Fast Hours, 2025-04-22

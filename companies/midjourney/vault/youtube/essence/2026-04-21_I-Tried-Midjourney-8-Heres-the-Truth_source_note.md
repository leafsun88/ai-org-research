# I Tried Midjourney 8... Here's the Truth

## V8 的官方卖点打在正确痛点上，但早期体验没有兑现“细节修复”的承诺

这条测试的价值在于它把 Midjourney V8 的承诺和创作者实际工作流放在同一张桌上：官方说细节跟随、style reference、文字、2K、五倍速度都变强，正好对应 Midjourney 过去被批评最多的手指、文字、画面一致性和输出质量问题。但 Caleb 的第一轮实测判断很负面，尤其是同一提示词下 V8 甚至不如 V7 稳。这不是一个简单的模型评分问题，而是 Midjourney 的产品风险：它最核心的用户心智是“审美强、能出概念”，如果 V8 没有明显修复细节，同时又削弱了旧版本的稳定感，创作者就会把它从成片工具降级为前期 mood board 工具。

“Midjourney 8 终于来了，团队说这是对我们用了很久的版本 7 的大更新和大改进，细节跟随更好，style reference 更强，图像更连贯更精细，文字渲染也好多了，生成速度快五倍，还加入了原生 2K 的 HD 模式；问题是 Midjourney 过去几个月处在一个奇怪位置，审美上仍然很强，很适合把概念变成电影感画面，但细节会变得很怪，手指太多、文字乱七八糟，各种奇怪东西都会冒出来，而 V8 的承诺就是解决这些问题。” —— Caleb / Curious Refuge, 2026-04-21

## 同一提示词下 V8 输给 V7，会让“升级”变成创作者的迁移成本

Caleb 用蓝色机甲、未来市场、蓝色宝石这类 Midjourney 强项提示词做对比，本该给 V8 展示电影感和视觉想象力，结果他认为 V8 输出质量明显更差。这类失误对 Midjourney 尤其伤，因为它不是在复杂文字排版或精确商业图里失败，而是在自己传统优势区失败。对创作者来说，版本升级不能只是“需要重新学习新 prompt”，还必须有足够强的性能回报；否则旧 prompt、旧 mood board、旧 style reference 的资产价值会下降，用户会延迟迁移。

“在旧版 Midjourney，也就是版本 7 里，我们得到的画面审美上很不错，基本就是我们提示词要的东西，看起来很酷，很有电影感，我喜欢虚化细节和调色；但在 Midjourney 8 里，我们得到这张图，我立刻就困惑了，因为质量比 Midjourney 7 差太多了，我不太确定这里发生了什么，但在这个具体例子里，版本 8 真的没有做得很好。” —— Caleb / Curious Refuge, 2026-04-21

## Style reference 是 Midjourney 的关键入口，但 V8 在可用性上反而受限

Caleb 的 workflow 代表专业创作者的真实路径：很少只靠一条 prompt 出最终图，而是先用参考图锁定调色、构图和摄影语言，再生成项目视觉。V8 如果宣称更擅长 style reference，就必须在这个路径里更可靠。但测试里 HD 与 Q4 不能同时使用 style reference，基础 V8 加参考图的结果又偏软、细节不稳，这让高分辨率和风格一致性变成互相排斥的选择。对广告、电影、平面设计工作流来说，这种限制会把 Midjourney 留在探索阶段，而不是交付阶段。

“你几乎总是会用某种 style reference，把生成推向某个具体方向，通常你会有一张图或一组图，体现你想要的调色、构图和整体 vibe；在 Midjourney 8 里做同样的事，只要输入提示词，再把原始参考图作为 style reference 放进去就可以了，但我想提醒一点，你不能在 HD 模式或 quality four 模式里使用 style reference，所以必须用基础版 Midjourney 8 来渲染。” —— Caleb / Curious Refuge, 2026-04-21

## Nano Banana Pro 抢走了“细节和文字”的工作，Midjourney 只剩审美探索护城河

这条视频把 Midjourney 和 Nano Banana Pro 的分工讲得很清楚：Nano Banana Pro 的 raw prompt 可能有 B+ stock photography 的模板感，但在文字、结构、参考图延展、细节可靠性上更像成片工具；Midjourney 的优势是初看更有风格、更有创意，但细看容易崩。这个竞争信号很关键，因为 Midjourney 如果不能把审美优势转成可控、可编辑、可交付的 workflow，它会被用户当成上游灵感发生器，而后续编辑、修复、落地交给 Google 或聚合平台。

“如果你眯着眼快速看，Midjourney 8 出来的图比 Nano Banana Pro 更好，质量似乎更有创意、更有意思；但当你真的开始看细节，它就全散掉了，所以你不能依赖 Midjourney 给你专业平面设计项目所需的高分辨率细节。短答案是，我认为 Midjourney 8 在创意项目里的实际效用大约和 Midjourney 7 差不多，你很可能不会直接用 Midjourney 生成进入电影项目或广告项目的最终图。” —— Caleb / Curious Refuge, 2026-04-21

## Midjourney 的最佳位置正在变成 mood board 和 style reference 生产机

Caleb 最后给出的定位不是“弃用 Midjourney”，而是重新定义它在创作链条里的位置：用 Midjourney 生成 mood board 和 style reference，再把这些素材带到 Nano Banana Pro 之类更强的模型里做精修。这对 Midjourney 是双刃剑。一方面，它保住了审美发现和视觉方向定义的价值；另一方面，它意味着用户付费理由从“完成作品”变成“产生前期灵感”，平台的使用频次、专业粘性和价格承受力都可能被下游工具重新分配。

“我确实认为 Midjourney 对生成 mood board 和 style reference 非常有价值，你可以把这些参考拿到像 Nano Banana Pro 这样的聪明图像生成器里使用；这最终落在 Midjourney 和许多正在出现的图像生成器之间的差异上，Midjourney 更像 diffusion-based 的图像生成工具，你可以想象把一堆颜料扔到画布上，再让系统把颜料抹开，直到大致得到你要的东西。” —— Caleb / Curious Refuge, 2026-04-21

## 模型架构差异被创作者体感成“想象力强但世界模型弱”

Caleb 用“颜料”和“乐高”解释 Midjourney 与 Nano Banana Pro 的差异，虽然不是严格技术论文语言，但很能说明创作者侧的产品感受：Midjourney 擅长从噪声里抹出有 vibe 的画面，但在物体关系、文字、局部细节、可控构图上像缺少更强世界模型；Nano Banana Pro 更像先理解场景结构再搭建画面，因此更可靠但也更容易模板化。Midjourney 下一阶段的关键不是单纯更漂亮，而是把“style”与“scene understanding”合在一起。

“如果和 Nano Banana Pro 这类 diffusion transformer 工具对比，它们更像乐高，有一些预定义资产知道必须进入最终图像；如果提示一个男人喝咖啡，它已经大致知道男人喝咖啡应该是什么样，也有智能去创建图像里的轮廓。问题是它会更泛、更没有创造性的风格，因为它对东西应该长什么样有预设，所以你几乎总是需要给 Nano Banana Pro 这类工具加入创意 style reference，才能获得最大质量。” —— Caleb / Curious Refuge, 2026-04-21

## 对 Midjourney 的核心警告是：竞争更新速度可能把它边缘化成辅助工具

视频最后的投资含义很直接：Midjourney 不是没有价值，而是窗口期正在被压缩。Google 这类对手一旦把风格和细节补齐，Midjourney 的独特性会从“最好的图像生成器”滑向“好用的风格灵感库”。这解释了为什么 V8 早期反应会重要：它不只是一次模型更新，而是在验证 Midjourney 能不能跟上从审美模型到可控创作系统的行业迁移。

“在 AI 图像生成世界里，Midjourney 的时钟已经开始倒计时了；他们真的需要更新自己的 world model，让它对正在发生的事情有更整体的理解，否则我认为 Google Nano Banana Pro 这类工具只要再有几个小更新，Midjourney 在创意语境里的帮助就会变得很有限。今天它还有帮助，但我能看到几个月后，随着行业里所有创新推进，它真的可能被边缘化。” —— Caleb / Curious Refuge, 2026-04-21

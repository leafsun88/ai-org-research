# Midjourney Fast Hours Ep.64 — Source Note

## V8 Alpha 最大的负面信号，是老用户没有像过去那样第一时间冲进去

这期节目最有价值的不是某一张图好不好，而是两位重度 Midjourney 用户对 V8 的情绪温度已经变了。过去 Midjourney 每次发模型都像打开礼物，创作者会马上跑进去测试；这次他们承认自己没有立刻冲，部分原因是等待太久，部分原因是其他工具已经进入工作流。对 Midjourney 的投资/产品判断来说，这是成熟期信号：品牌仍有爱和肌肉记忆，但版本发布不再自动制造不可替代感。

“我没有像过去每次发布那样马上冲进去试；以前那感觉像打开圣诞礼物，像等了很久的包裹终于送到门口。但这一次很多人的 workflow 已经变了，其他工具已经进入我们的工作流，我们等 V8 等了太久，所以某种程度上我们已经移动了。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-21

## Alpha 站点发布把“新模型一定更好”的默认期待打破了

主持人的失望集中在“等了一年却拿到 alpha”这个节奏上，而不是单纯抱怨画质。V7 到 V8 的等待期被他们感知为接近一年，外部竞争又在加速，因此 alpha 版本的不稳定会直接伤害非 power users 的第一印象。Midjourney 过去可以让用户花时间重新学习一个模型；现在市场上有 Nano、ChatGPT image、Reve、Runway/Flora/Weavy 这类替代路径，发布质量本身变成留存机制。

“我失望的第一件事是它还在 alpha 站点上。上一个模型 V7 是 2025 年 4 月出来的，现在已经到 2026 年 3 月，我们等了这么久，拿到的是一个 alpha？在现在这种竞争环境里，很多非重度用户进去一试，发现它没有按新模型应该有的方式完全工作，就会马上失望。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-21

## Mood boards 在 V8 里的退化，是创作者工作流里最真实的痛点

两位主持人反复测试 SV6/SV7、单 mood board、多 mood boards 和 V7/V8 对比，结论不是 V8 没有亮点，而是 mood boards 这个曾经强绑定个人风格的机制在 V8 上不稳定。对 Midjourney 来说，mood board/SREF/个人化不是边缘功能，它们是老用户积累风格资产和持续付费的核心原因；如果这部分质量下降，用户会暂时退回 V7 或转向节点式工具。

“如果我现在要用 mood boards，我不会在 V8 里用，至少在进一步修好之前不会；我们把同样的 mood boards 放回 V7，结果马上好十倍，很多图都可用。V8 里有些图有趣，但 mood board 的退化是真实存在的，这很可能就是他们还在修的东西。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-21

## V8 的 prompt 方式从“短而有 vibe”转向更结构化，改变了 Midjourney 的娱乐属性

节目里反复出现一个微妙变化：V8 更听细节，但不再像 V7 那样靠短 prompt 和 mood board 就能爆出“banger”。这会改变产品的默认使用心智。Midjourney 原本很强的一点是用户可以像玩一样探索模型的审美潜意识；V8 如果要求更多 camera shot、lighting、environment、medium 这类结构化描述，就会更接近专业生产工具，同时削弱“随手 vibe create”的黏性。

“V8 对 prompt 的反应和 V7 不一样，短、vibey 的东西已经不再自动带你到那个结果；也许 JSON prompting 或更结构化的方式会更适合它，你可能需要写清楚镜头、摄影风格、灯光、环境、媒介这些东西。以前我可以只写‘the work is dark and mysterious and important’就拿到好图，现在它不一定这样工作了。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-21

## Stylize 和质量参数的边际差异变小，会让复杂参数系统显得更重

他们在 V8 里测试 stylize 150 到 900、Q4/HD、raw/standard 等变量时，一个重要观察是参数谱系的感知差异没有过去那么大。对 Midjourney 这种参数密集型产品而言，这很关键：如果 slider 数字看起来从 0 到 1000，但结果差异并不对应，用户会感觉系统复杂却不可解释。长期看，Midjourney 可能需要把某些参数合并成更少、更清晰的模式，而不是继续叠加开关。

“我现在没看到 stylize 从 100 到 1000 有巨大跳跃，200 和 800 的差别没有以前那么夸张；如果现实里这个 spectrum 没有那么宽，那也许就不该让用户面对整个 slider，假装这里有一个很大的零到一千的区别。也许可以把它压成几个停靠点，让选择更清楚。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-21

## V8 底层仍有强审美能力，问题是稳定性和可控性还没有跟上

节目不是单向否定 V8。两位主持人在无 personalization 或简单 prompt 下看到更真实的构图、更自然的裁切、非对称负空间、低角度近景、摄影感焦点等，这些是其他模型很难自动给出的 Midjourney 特长。这里的判断应该拆开：Midjourney 仍然有独特视觉模型能力，但创作者要把它用于项目时，需要稳定、编辑、reference、upscale 这些控制层跟上。

“这里面有些东西真的很好：有些构图你在别的模型里根本不会自然得到，比如人物偏在左侧、头顶没有太多负空间、胡子被底部裁掉、眼睛没有直视镜头、hood 部分落在背后。除非你在其他平台里把每个镜头细节都工程化描述，否则很难得到这种真实、不摆拍的感觉。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-21

## 编辑模型是 V8 之后真正决定用户是否回流的功能

主持人明确把 edit model 放在 V8 后最重要的位置，因为多步骤项目和商业工作流依赖的是修改、保持角色、局部替换、从 still 到更多资产，而不是一次生成漂亮图。Midjourney 的发布策略如果继续“先丢 alpha 再让用户反馈”，在 edit model 上会更危险，因为用户已经习惯 Nano/Freepik/Luma 等工具的一键编辑和 agentic sequence。编辑模型不是附属能力，而是从 inspiration 工具进入 production workflow 的门票。

“等你把 edit model 给我，我才知道我对 Midjourney 这条线到底有多 bullish；对我们这种做多步骤项目工作的人来说，编辑模型是主要决定因素。现在已经有 Freepik 和 Luma agents，可以一张图一键生成八张连续图、同一个角色讲一个故事；我不会再花时间 hack 一个不好用的 edit model。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-21

## Midjourney 过去靠用户学习新模型，现在这个 playbook 被替代工具削弱了

节目里有一句非常强的社区判断：过去每次 Midjourney 新模型不顺，重度用户会说“你还没学会它的新架构”；现在他们自己也承认，用户没有同样的时间和耐心。这个变化对 Midjourney 的社区护城河很重要。它仍有深度用户，但“花几周学会控制新模型”不再是所有创作者愿意支付的隐性成本，尤其当 LLM/image hybrid 工具能直接听自然语言编辑时。

“我以前也会说，新模型刚出来大家都不喜欢，然后花时间就能学会新的 prompting style；但那是以前没有别的选择的时候。现在我没有那么多空闲时间去学习怎么控制 Midjourney，尤其当我知道它还做不了某些我需要的事时，我就必须把时间花到别处。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-21

## Claude Code 让创作者开始自建工具，进一步抬高了 Midjourney 的 workflow 门槛

后半段看似转向 Claude Code，但对 Midjourney 研究很有用：这些同一批视觉创作者已经开始用 Claude/Codex/Notion/Weavy 自建 Chrome extension、下载器、prompt database、style forensic pipeline、Calendly 替代品和批量素材处理系统。Midjourney 面临的竞争不只是另一个 image model，而是用户开始把创意生产流程拆成可编排的本地/节点/agent 系统。Midjourney 如果只保持“生成漂亮图”，会被嵌入别人的 workflow，而不是拥有 workflow。

“Claude Code 是发生在我身上最好也最糟的事；最好是因为它让我把事情做完，最糟是我被‘现在能做什么’完全吞掉了。我现在可以做 Chrome extension、Pinterest alt text、批量下载 board、分析 800 张图的风格、把它变成 system prompt，再放进 Weavy 或品牌内容 workflow 里；这些以前都要花很久，现在像积木一样不断被我接起来。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-21

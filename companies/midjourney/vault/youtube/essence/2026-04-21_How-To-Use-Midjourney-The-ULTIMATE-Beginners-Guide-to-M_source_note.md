# Midjourney / Wes Roth 2026-04-21 — Source Note

## Midjourney 的新手入口已经从 Discord 迁到网页创作台，用户第一步变成“打开 Create 并输入 prompt”

这条教程的研究价值在于它默认用户从 `midjourney.com` 和网页端 `Create` 开始，而不是从 Discord bot 开始。对 Midjourney 来说，这意味着 onboarding 的叙事已经从“加入一个社区服务器、学 slash command”转成“打开一个创作产品、输入 prompt、看模型把模糊画面变清晰”。这种入口变化会降低早期 Discord 门槛，但也把产品教育压力转移到网页端控件：用户必须理解 prompt、四图结果、vary、upscale、animate、参数和 pricing 的组合。

“要开始使用，先去 midjourney.com，登录，然后打开 Create，就是这个小图标。你在这里输入 prompt，然后按回车；我输入的是一个 origami 风格的 John Wick 战斗场景，系统会开始创建你脑子里想象的东西。你会看到模型好像从很模糊的东西里把图像带出来，慢慢变成一个看起来不错的结果，然后你可以点开每张图，或者用方向键在四张结果之间切换。” —— Wes Roth, YouTube, 2026-04-21

## Vary、Upscale、Animate 把一次生成改成分叉式 workflow，创作动作不是“拿结果”，而是连续挑选和再计算

教程把 Midjourney 的核心使用方式讲成一个迭代树：先选四张初稿，再在 `V subtle`、`V strong`、`upscale subtle`、`upscale creative`、`animate`、`extend` 之间不断分叉。这里的 adoption insight 是：Midjourney 的用户价值不只来自模型一次性出图，而来自一套低门槛的 search interface。用户不需要会画图，但需要会判断哪条分叉值得继续投入 GPU compute；这也解释了为什么 pricing、fast hours 和 relaxed mode 会直接影响创作节奏。

“下一步基本有两个选择：要么 vary，要么 upscale。V subtle 的意思是我喜欢这个概念，请给我更多类似的，只做小变化；V strong 则是给我更多类似的，但变化要大。你可以一直这样迭代下去，不断做 subtle variation、创造新图，直到找到你想要的那个完美感觉；如果你已经满意了，就点 upscale，它会生成更大、更高分辨率的版本，而且我发现原图里一些奇怪的地方通常也会被平滑掉。” —— Wes Roth, YouTube, 2026-04-21

## 视频功能把 Midjourney 从图像工具推向素材生产工具，但成本教育必须前置

Wes Roth 特别解释视频 pricing，说明视频功能会把 Midjourney 推向商业 B-roll、短片、素材批量生成这类更高频的 workload。关键不是“能不能生成视频”，而是用户能否理解 `fast` 和 `relaxed` 的经济学：Pro/Mega 用户可以用 relaxed mode 做无限视频，但要接受排队延迟；商业收入超过一定规模还必须进入更高套餐。这是产品商业化的一个小窗口：Midjourney 以 compute queue 管理成本，用等待时间换无限量使用，把专业创作者从按次焦虑里释放出来。

“我想很快讲一下定价，尤其是视频，因为这里有些东西你需要知道。Midjourney 有 relaxed mode：你输入 prompt 后，fast mode 会立刻给结果，而 relaxed mode 会等 GPU 更空的时候再做；每个月你的 fast hours 是有限的，但 Pro 和 Mega plan 有无限 relaxed mode video。如果你想要无限视频，可以有，只要用 relaxed mode，它可能要等十分钟才开始，但你可以做无限视频；如果你是商业用途，订阅者大体上被允许使用，不过年收入超过一百万美元就必须用 Pro 或 Mega。” —— Wes Roth, YouTube, 2026-04-21

## Explore 页承担了用户教育功能，社区好作品直接变成 prompt 学习材料

教程反复建议新手去 Explore 看社区最受欢迎的图片和视频，并直接阅读别人的 prompt。这说明 Midjourney 的 learning loop 很大一部分不是正式文档，而是公开创作流：用户通过拆解“好结果背后的 prompt”学习 medium、lighting、shot type、style 和参数。对组织研究来说，这是一种社区产品化的能力：Midjourney 把用户作品、prompt 和偏好聚合成持续的教育资产，降低新手从空白 prompt 到可控输出的距离。

“问题是怎样变得更好？去 Explore section，它会展示社区这一周、今天或这个月最受欢迎的图片和视频。你可以看大家都在做什么；我非常建议你去看 top week、top month、top day，那些你真的喜欢的风格就点进去，你会看到他们实际用了什么 prompt。你能通过看这些 prompt 和结果，开始理解不同写法怎样转化成最终图像。” —— Wes Roth, YouTube, 2026-04-21

## 新手 prompt 教育的核心不是神秘咒语，而是让用户掌握摄影和视觉语言

这条教程把 prompt 写作拆成 subject、medium、lighting、mood、shot type、era 等普通视觉语言。Midjourney 的 adoption 难点因此不只是软件操作，而是把非专业用户训练成能说清视觉意图的人。`stock photo`、`GQ magazine`、`macro photography`、`airbrush painting` 这些词不是参数，而是用户把脑中风格压缩成模型可理解 signal 的方式。换句话说，Midjourney 的用户教育会持续外溢到摄影、美术、设计语言的普及。

“在 prompt 里，要具体说明 subject、medium、lighting、mood、shot type、era 等等。medium 可能是最重要的：你想要 pastel painting、pixel art、真实相机拍摄、macro photography，还是别的东西？最好的输出通常都会指定一种 medium。比如 stock photo 这个词会让结果像一张那类主题的库存照片，GQ magazine、白色背景、脸部对比、realistic photography，也会给模型一个现代杂志拍摄的方向。” —— Wes Roth, YouTube, 2026-04-21

## Personalization 和 mood boards 把一次性 prompt 变成长期审美 profile，Midjourney 在向“个人风格系统”移动

教程最后提到 personalization 和 mood boards，这部分比普通参数更有战略含义。Midjourney 不只是让用户每次写 prompt，而是让用户通过偏好选择、图片集合和风格板，把自己的 taste 持久化。对专业创作者来说，这会让 Midjourney 从单次生成工具变成风格一致性的 workflow；对公司来说，用户偏好数据和 style consistency 会成为留存层，而不是只靠模型质量竞争。

“有些很酷的东西值得试，比如 personalize：你可以打开 personalization，然后选择你喜欢哪些图。随着时间推移，AI model 会为你建立一个关于你喜欢什么的 profile，并尽量把结果 stylize 到你偏好的风格。最新的还有 mood boards，我还没太玩过，但如果你有一个具体 vision，比如做一个风格一致的卡通，你可以 curate 一组图片并设定 style；focused mood boards 会创造 consistency。” —— Wes Roth, YouTube, 2026-04-21

# Midjourney Fast Hours Ep.62 — Source Note

## V8 延迟让社区重新关注“发布 cadence”，而不是只等下一次大模型跃迁

这期是在 V8 还没出来时录的，价值在于捕捉老用户对等待的疲劳。主持人记得 V8 从上一年末、1 月一路被预期到 2 月，结果仍未到来；这种反复延后会把用户注意力从“模型会变多强”转向“公司能不能按节奏 ship”。Midjourney 过去靠大版本制造兴奋，但现在用户明确希望更稳定的小步迭代、质量更新和功能回归。

“我们还没有 V8，所以不要把现在的 quality of life 更新当成 V8；但我们之前就说过，V8 从十月、十一月开始被讨论，原本感觉可能年底或一月来，现在一月已经过去，下周就二月了，还是没有 V8。这种感觉很熟悉，我们当然准备好了，但也真的在等。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-22

## Mood boards 和 style creator 是老用户最希望随 V8 一起升级的风格资产层

主持人不是只盼望更高画质，而是特别盯着 mood boards、style creator 和 personalization，因为这些功能承载了用户的个人风格积累。V7/V8 模型更新如果没有把这些资产顺利带过去，就会破坏重度用户的长期复利。这里能看到 Midjourney 的差异化不只在模型输出，而在用户把自己的 taste 变成可复用风格系统的能力。

“我前几天又重新玩 mood boards，我真的很想看到 V8 在这里有跃迁；不管你放进去的图片多高清，用 mood boards 创建时总感觉质量会降一点，也许和它的架构有关。Mood boards 加上 style creator 是最快创造非常个人化、独特、能规模化输出的方式，我希望 V8 把这部分也解决。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-22

## Midjourney 仍然是最适合“文字触发审美联想”的工具

节目里有一段很能解释 Midjourney 的品牌护城河：用户会把奇怪词、公司黑话、歌词、数学结构、短语扔进去，看模型怎样解释。这种玩法和 Nano/ChatGPT image 的自然语言执行不同，它更像和一个视觉潜意识系统玩。Midjourney 即使被批评落后，仍拥有“prompt as aesthetic exploration”的独特位置；这也是为什么用户对它失望但还想回来。

“Midjourney 是我唯一喜欢 text prompt 的地方，因为我知道这些词放到别处不会这样工作；我可以把一个怪词、一个公司短语、一个歌名或者数学结构扔进去，看模型怎么解释。像 circle back、course correct 这种企业黑话，它会变成某种 severance 式的恐怖 corporate visual，这种魔法是我还会回来的原因。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-22

## V8 必须缩小“可玩图”和“业务可交付图”之间的质量差距

主持人对 V8 的期待从漂亮升级到商业可用：少 artifacts、少解剖错误、更好皮肤纹理、物理一致、能用于 client project。这个判断很适合写产品策略，因为 Midjourney 的早期优势是美和惊喜，但下一阶段创作者要把图交给客户、品牌、广告和内容系统，容错率就降低。Nano Banana Pro 被拿来当对照，因为它在产品一致性、contact sheet、物理正确性上更像 production 工具。

“我现在更关心的是把小 artifacts、hallucinations 和物理不一致都清掉；如果你要把图用于真实 client project，它必须更 tight，必须达到 business project ready 的程度。Nano Banana Pro 现在很疯狂，你让它给同一个产品做 12 张不同角度 contact sheet，它就直接 nail，所以 V8 必须把这个 margin 缩小。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-22

## 新 personalization UI 的目标是降低认知负担，但用户疑惑为什么不复用 Explore 行为数据

主持人提到 Midjourney 正在测试新的 personalization interfaces，目标是减少 binary A/B ranking 的认知负担、一次展示更多图、加快偏好选择。这个方向本身正中痛点，因为 rank 图对新老用户都累；但他们也提出一个产品问题：Explore 页面的 likes/选择是否本来就能成为 personalization signal。Midjourney 的挑战是把用户自然行为转成偏好模型，而不是把偏好训练做成额外任务。

“新的 personalization interface 在测试，目标是比二元 A/B ranking 减少认知负担，一次展示更多图，让偏好选择更快；这听起来像给 Explore page 打开 personalization 开关。以前他们也说过 Explore 上你喜欢的东西会起一点作用，所以我不太懂为什么不直接走那条路，当然我不知道后台怎么做。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-22

## 编辑模型是 Midjourney 当前最重要的回流杠杆

这期在 V8 之前就已经把 edit model 置于核心位置。主持人认为传统 inpainting 和 Nano Banana 式自然语言编辑可能并存，而自然语言编辑会影响 coherence 和 model architecture。更重要的是，他们明确说 Midjourney 要让用户尽可能留在工具里，因为现在竞争太多，而且这是他使用 Midjourney 最少的一段时间。编辑模型决定 Midjourney 是独立灵感工具，还是继续占据日常 production loop。

“编辑模型可能是现在最重要的东西，真的；如果我是 Midjourney，我会想着尽快推 V8、尽快推 edit model，并且想办法让人尽可能多、尽可能久地留在工具里，因为现在竞争已经来了。这是我花在 Midjourney 里最少的一段时间，而我其实是想待在 Midjourney 里的。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-22

## Nano Banana / Gemini 的 LLM+image 模式正在重塑创作者对 prompt 的理解

节目中部对 Nano Banana/Gemini 的讨论很关键：主持人发现 image model 不只是执行 prompt，而是可以先联网搜索、识别产品、读取评论、理解真实物件，再把上下文应用到图像。用户开始把 prompt 写成多步骤系统指令，而不是 Midjourney 式视觉描述。这是 Midjourney 面临的结构性竞争：LLM-native image workflow 能吃掉 research、editing、brand guardrails、system prompt 和执行，Midjourney 必须决定自己是否也往这个方向走。

“Nano Banana 不只是执行工具，它背后连着 LLM；你可以让 Gemini 看到一张刷子的图，去网上找这个刷子、找 Amazon、找评论，再把真实世界上下文应用到图像里。也就是说 prompt 不是单纯一句生成指令，你可以给它研究、确认、步骤和规则，像在 LLM 里一样，只是最后产出的是图片。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-22

## 创作者工作流的 skill gap 正在从“会 prompt”升级为“会组织数据、风格库和 system prompt”

这期标题里的 skill gap 不只指会不会用 Midjourney，而是 AI 创作进入更复杂的操作系统：用户用 Cosmos/Pinterest 收集图，用 Claude 分析风格，用 Google Drive/Fireflies/Weavy 管理 prompt library 和 workflow，用 system prompt 复用品牌视觉规则。Midjourney 的机会是成为这些流程里的高质量生成核心；风险是只成为其中一个节点，被更会组织 workflow 的工具层包住。

“现在关键变成数据集和组织方式：我会从 Cosmos 或 Pinterest 拉一组图，导出文件夹，让 Claude 本地分析每张图的风格规则，提取负空间、角度、色彩、材质、logo 放置方式，再把这些变成 system prompt。你花时间的地方不再只是写 prompt，而是 curated dataset、style guide、prompt database 和可复用 workflow。” —— Rory Flynn / Drew Brucker, Midjourney Fast Hours, 2026-04-22

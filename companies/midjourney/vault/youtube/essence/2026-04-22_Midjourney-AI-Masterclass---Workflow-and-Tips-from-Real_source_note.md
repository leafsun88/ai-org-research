# Midjourney AI Masterclass — Source Note

## 真实客户案例证明 Midjourney 已进入付费设计服务，而不是停留在概念演示

Stephen Gates 开场就把视频定位成反理论化：他经营设计工作室 Crazy，过去两年用 Midjourney 和相邻工具给真实客户做 branding、advertising、digital experience、product 和概念内容。他举 Mary Dowling whiskey、cocktail program、event booth、Metallica 等案例，强调客户预算变紧、创意需求更高，AI 让小工作室能产出以前预算不支持的视觉。这个 source 是 Midjourney 专业采用证据里最直接的一条。

“我们不是在讲 Batman pop-up store 或 Iron Man 变成武士这种理论展示，我们是一家真实设计工作室，过去两年用 Midjourney 和其他工具给真实客户做真实工作；预算越来越紧，客户还想要更多创意，所以我们用这些工具创造他们以前根本没有预算生产的高端产品视觉、广告、活动和品牌内容。” —— Stephen Gates, The Crazy One, 2026-04-22

## AI 不会直接拿走工作，但会把竞争基线抬到使用 AI 的人那里

Stephen 的职业判断很硬：AI 目前不会大规模替代设计师，因为客户仍然很难清楚表达他们要什么；但会用 AI 的设计师、工作室和竞争者会替代不会用 AI 的人。这个判断解释 Midjourney 的市场采用路径：不是 end customer 直接绕过设计师，而是专业服务供给侧先吸收 Midjourney，变得更快、更便宜、更能展示可能性，继而重塑客户预期。

“现在 AI 还不会拿走你的工作，但现实是会用 AI 的人几乎肯定会；对客户来说，要在大规模上用 AI 替代我们，他们还得非常清楚地表达自己想要什么，所以短期还安全，但这个行业的拐点已经来了。” —— Stephen Gates, The Crazy One, 2026-04-22

## Studio 工作流被压成 create、refine、finish、repeat、animate、evolve 六步

Stephen 把 Crazy 的 Midjourney 流程概括成六步：create、refine、finish、repeat、animate、evolve，其中 repeat 是最能拉开专业差距的环节。这个结构很适合后续分析 Midjourney 如何进入 production workflow：生成只是第一步，真正商业价值来自修正、后期、系列化、动画化和持续演化。Midjourney 的产品位置是整个流水线的起点和回流点。

“我们现在的流程基本是六步：create、refine、finish、repeat、animate、evolve；其中第四步 repeat 是关键，如果你能把它做好，就能和其他人拉开差距，因为客户通常不会只需要一张图，他们需要一整套能讲故事、能重复生产的资产。” —— Stephen Gates, The Crazy One, 2026-04-22

## 结构化 prompt 不是玄学，而是为了让团队可重复地产出一致结果

Crazy 为不同平台建立 prompt formula，内容包括 medium、subject、negative parameters、environment、time of day、composition、action、style、camera、lens、lighting 等。Stephen 的重点不是写得复杂，而是写得一致，方便团队更快得到可控结果。这是 Midjourney 从个人玩具变成组织工具的关键：当 prompt structure 可训练、可复用，工作室内部就能把经验传给更多设计师。

“我们给不同平台做了不同公式，写图像 prompt 时会从 medium 开始，再写 subject、环境、时间、构图、动作、风格、相机、镜头和光线，也会写不要什么；重点是养成一致写法，这样你更快得到一致结果，不会每次都花几个小时坐在那里碰运气。” —— Stephen Gates, The Crazy One, 2026-04-22

## Web interface、describe、shorten 和 aspect ratio 让 Midjourney 变成可学习的专业工具

Stephen 认为 Midjourney web interface 是 game changer，但仍会回 Discord 用 describe 和 shorten，因为它们帮助用户理解模型如何看图、如何权衡 prompt。Aspect ratio 和 model version 也会强烈改变输出：同一提示在方图、宽图、竖图里会被不同构图，在 V4、V5、V6.1 中也有完全不同气质。这个细节说明专业采用不是单纯买订阅，而是建立对模型参数和 UI 的操作知识。

“我每天都会看 Midjourney 首页学习别人的 prompt，也会回 Discord 用 describe 让模型告诉我它看见了什么，用 shorten 看它怎样处理我的提示；很多时候 prompt 是对的，aspect ratio 错了，换成 16:9 才能得到大景，换成竖图才会拉近人物，模型版本也会把同一个 prompt 变成完全不同的风格。” —— Stephen Gates, The Crazy One, 2026-04-22

## Midjourney image editor 把大修图从 Photoshop / Firefly 拉回原生模型

Stephen 说两年前他们很多工作还在 Adobe Photoshop、Firefly 和 generative fill 里做，但 Midjourney 的 image editor 改变了这点。原因是图像本来就来自 Midjourney，引擎更理解原图语境，扩图、重构 framing、擦除、局部改动往往比外部填充更自然。这个产品功能让 Midjourney 更接近 production tool，而不只是生成器；它减少了在外部工具间来回切换的摩擦。

“几个月前 Midjourney 发布 image editor 后，我们的流程变了；现在通常先用 prompt 和 image ratio 得到构图和 look，再用 editor 调 framing、修细节、擦除或扩展画面。因为图像本来就从 Midjourney 来，它补出来的结果远远好过我们以前用 generative fill 得到的东西。” —— Stephen Gates, The Crazy One, 2026-04-22

## 商业级输出仍依赖 Magnific、Gigapixel 和传统后期把 AI 图变成可交付资产

Stephen 讲得很实际：Midjourney 生成的图通常还不够真实、不够高分辨率，最多适合数字渠道；要用于客户工作、广告、billboard、stadium side 等，需要 Magnific 增强真实质感、Gigapixel 做高分辨率输出，再用 Firefly / generative fill 修 artifact。这说明 Midjourney 的客户价值不等于原始输出，而在一个 finishing stack 里才能变成 production-ready asset。

“Midjourney 会给你一张好图，但很多时候它还需要后处理和 upscale 才能真正用于客户工作；Magnific 对我们是巨大 unlock，它不只放大，还会增强头发、皮肤质感和真实感，之后 Gigapixel 能做极高分辨率输出，如果要上 billboard 或体育场侧面，这些最后一步就很关键。” —— Stephen Gates, The Crazy One, 2026-04-22

## Repeatability 是客户愿意为 AI 视觉付费的核心，而不是第一张图有多漂亮

Stephen 说很多大 agency 或艺术家找到他们，是因为自己做出一张喜欢的 AI 图后，不知道怎么变成 campaign。Crazy 的经验是，任何人花点时间都能做出一张不错的图，难的是做 12 张、做一组照片、做一个完整 campaign，并且看起来像同一个 expensive shoot。这个判断直接击中 Midjourney 的 professional moat：真正高价值用户在训练风格、管理 prompt、复用 reference，让结果成系列。

“我们甚至会被大型 agency 和艺术家雇佣，因为他们做出一张喜欢的图后说太好了，把它变成 campaign 吧，然后发现自己不知道怎么做第二张；做一张不错图不是火箭科学，真正的 trick 是做 12 张，做一整个 campaign，让它们像同一个昂贵拍摄项目，而不是一堆 AI one-offs。” —— Stephen Gates, The Crazy One, 2026-04-22

## Character reference、style reference、image prompt 和 prompt sheet 让真实人物品牌资产可管理

Mary Dowling 案例展示了 Midjourney 在历史人物和品牌叙事里的用法。团队用真实照片做 character reference，控制 stylize 强度来决定像本人、年轻一点还是更宽泛；同时叠 style reference 保留 turn-of-the-century 质感。为了后续复用，他们把 cocktail program 的 prompt、训练图、personalization model、stylize、model、raw style、aspect ratio 等存进 Google Sheet。这是非常具体的组织化采用证据。

“做 Mary Dowling 时，我们用真实照片作为 character reference，因为我们是在代表一个真实女性；如果 stylize 设为 0，会更像精确复制，100 像本人但可能年轻一点，200 像表亲，300 以上更宽泛。我们还会同时用 style reference 保留世纪之交照片感，并在 Google Sheet 里记录每个 cocktail prompt、训练图、模型、比例和设置，方便以后再做。” —— Stephen Gates, The Crazy One, 2026-04-22

## AI 视频对客户工作仍不稳定，成本和失败率决定了采用节奏

Stephen 对 video generation 的判断比图像保守。他们会用 Runway Gen-3 Alpha 和 Luma 处理 Midjourney 图像，但客户工作里失败率高、prompt 重跑次数多、并发受限、unlimited plan 每 seat 数百美元，而且每个可用视频背后可能有一百个不可用输出。这是很好的反方证据：Midjourney 图像生产已进入客户工作，image-to-video 正在进入，但商业级稳定性还没完全到位。

“如果只是做酷实验，视频工具已经很好玩；但如果是真正客户工作，现在仍然让人非常沮丧。一个可用视频背后可能有一百个像橙汁瓶一样莫名其妙的结果，而且即使 unlimited plan 每 seat 要几百美元，也常常只能同时跑三四个视频，所以我会等这块再成熟一点。” —— Stephen Gates, The Crazy One, 2026-04-22

## Proof of technology 改变客户协作方式，设计师要同时管理 AI、客户和伦理风险

Stephen 最后的职业判断很重要：他们会给客户做 proof of technology，快速展示 AI 能完成 cocktail program、concept video 或视觉方向，让客户和团队看到可能性；但目标不是为了 AI 而 AI，而是卖产品、改变业务、产生影响。专业设计师的新技能包括和 AI algorithm 协作、和使用 AI 的人协作、知道哪些交给 AI、哪些 human-centered design 必须保留，以及识别版权、商标、训练数据、客户 IP 和工具数据政策风险。

“我们做 proof of technology，是为了告诉客户和自己这些工具进化得很快，给我们几个小时就能试出可能性；但目标不是做 AI for AI's sake，而是做能连接消费者、突出品牌、卖产品、改变业务的东西。我们还必须理解版权、商标、训练数据、客户 IP 和数据政策，有些模型我们不会用，因为它们不说明训练来源或客户数据会发生什么。” —— Stephen Gates, The Crazy One, 2026-04-22

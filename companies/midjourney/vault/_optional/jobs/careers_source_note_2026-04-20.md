# Midjourney 招聘与组织补充笔记

## 核心来源

- Careers: https://www.midjourney.com/careers

## 已核验事实

- Midjourney 自称 non-traditional，features 由 community 建议和排序，projects 由 engineers 与 founder 领导。
- 公司强调 intentionally small team，以便更快迭代。
- 公司有 San Francisco、London、New York 办公室，起源于 remote，沟通大量发生在 Discord。
- 已披露核心 domain：research、infrastructure、frontend。
- Research 包括 diffusion models、reinforcement learning、training optimization、data processing。
- Infrastructure 包括 data center / cluster management、inference optimization、large scale cloud systems。
- Frontend 包括 web / mobile product design and engineering、new feature prototyping、internal tools。
- 公司提到还有 image generation 之外的 secret projects。
- 招人标准强调 self-directed、communicative、highly proactive；比起 resume/job history，更看 projects / portfolio；优秀候选人可能来自高中项目、知名公司或大学。
- Careers 页显示长期开放优秀 engineers / researchers，当前尤其需要 research 与 data engineering。
- 公开岗位包括 AI Researcher、Product Engineer (Web)、Product Designer、Data Engineer 等。Data Engineer 职责涉及跨系统数据整合、interactive dashboards、financial planning platform、automated insights。

## 研究含义

- Midjourney 的组织动作和 Lovable 有一个相似点：小团队、工程师直接 owner、快速迭代；差异在于 Midjourney 的反馈源更偏 community/aesthetic，而 Lovable 的反馈源更偏 work simulation / software production。
- 它不是典型企业 SaaS 组织，careers 里没有把销售、CS、deployment strategist、enterprise security 放在核心位置；更像 research + infra + frontend + community loop。
- Data Engineer 角色值得跟踪。它说明公司正在把财务、产品和社区行为转成可管理的内部仪表盘，可能是小团队维持大规模 consumer subscription 的关键补课。

## 限制

- Careers 页未披露组织架构、HR 负责人、完整管理层名单和员工数。
- 官网职位可能随时变动；本笔记只代表 2026-04-20 可见状态。

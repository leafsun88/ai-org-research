# Base44 on Vibe Coding and SaaS

## Maor 把 vibe coding 看成最大的 software category，前提是模型竞赛保持多强格局

Base44 创始人 Maor 的判断很适合给 Lovable 做外部校准：如果模型 provider 没有一家长期大幅领先，vibe coding platform 就有机会控制软件生产入口；如果某个模型 provider 突然赢下全栈，它就可能顺手吃掉 vibe coding 市场。这个前提很关键：Lovable 的上限依赖模型层继续竞争，模型价格下降、能力提升，应用层才能把更多价值转成 margin 和用户体验。

“If one model provider wins the market by a wide margin, the next logical thing is to conquer the vibe coding market, because this will be the largest software category. As long as there is a race, we are positioned to control a large share of the market and bring a better solution.” —— Maor Shlomo, 20VC, 2025-11-24

## Google 是 Base44 眼里最大的潜在威胁，因为它有 model、cloud、data 和 Workspace stack

Maor 认为最大威胁来自 Google：如果 Gemini 在模型上赢，它拥有 cloud、data、Workspace、integrations，可以把 vibe coding 直接放进企业生产力套件。这是 Lovable enterprise 线的关键风险。Lovable 可以先靠 simplicity 和 brand 起量，但长期要对抗的是大厂把生成软件能力嵌进现有工作流和数据系统。

“Google, by far, is the one I worry about. If Gemini wins the model race, they have the entire stack: Google Cloud, data, integrations, Google Suite, and everything they need to build an incredible empire. I am hoping for a tough race, and right now it looks like one.” —— Maor Shlomo, 20VC, 2025-11-24

## Vibe coding 公司的成本结构很敏感，但模型价格下降会持续改善 margin

Maor 对 margin 的态度比市场担忧乐观。他认为模型价格会继续下降，现有模型能力还远没有榨干经济价值。如果这个判断成立，Lovable 的早期高推理成本压力会随着模型层竞争缓解；真正的问题会转向用户留存、工作流深度和分发，毛利率只是一组变量之一。这个观点也支撑 Lovable 的 freemium/word-of-mouth 策略：先用体验换增长，后面吃模型成本下降红利。

“Margins are the thing I worry least about in the industry, because model prices will go down toward zero. I do not think we will hit a wall with LLM improvements. Even with existing models, we are only scratching the surface of the economic value you can create.” —— Maor Shlomo, 20VC, 2025-11-24

## Vibe coding 对 SaaS 的杀伤力，在于让软件从 one-size-fits-all 变成 tailored code

Maor 对 SaaS 的替代逻辑很具体：很多 SMB/部门软件不需要一个臃肿的通用系统，而需要一个更 lean、更符合自己流程、可以不断改的工具。LLM 可以把 software tails 做出来，让用户不用把数据交给多个 vendor，也不用被固定功能绑住。这个框架同样解释 Lovable enterprise 的吸引力：企业内部很多小 workflow 过去不值得定制，现在可以被自然语言生成。

“The current model of one-size-fits-all software does not make sense for many buyers. Software becomes more liquid. You can take a starting point, templates, or open-source projects and vibe code your own features. Small businesses will want something leaner, more customized, understandable, and adaptable to what they need.” —— Maor Shlomo, 20VC, 2025-11-24

## 模型供应商切换只是一行配置，Lovable 要把价值建在 workflow 上

Maor 提到应用层会在 Claude、Gemini、OpenAI 之间快速切换，甚至一行字符串就能把数百万美元 spend 转走。这说明模型 API 本身无法成为 Lovable 的防御，Lovable 的防御要来自用户 workflow、数据、体验、分发和业务结果。模型层会继续内卷，应用层赢家会把这场竞争变成自己的成本和能力红利。

“The market dynamics are insane. We spend a lot on AI, and in a split second you change one string in the code and move millions in spend to a different provider. It bounces back depending on who has the best model, cost, and throughput.” —— Maor Shlomo, 20VC, 2025-11-24

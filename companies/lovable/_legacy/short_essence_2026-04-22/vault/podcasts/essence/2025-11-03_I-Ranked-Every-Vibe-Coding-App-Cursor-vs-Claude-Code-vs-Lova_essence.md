# I Ranked Every Vibe Coding App

## 用户会把 team trust 纳入工具选择，Windsurf 的组织事件直接伤害产品信任

这条材料来自第三方用户讨论，但它提供了真实用户选择工具时的心理变量：AI coding tool 承接用户的项目、代码和创业进度，团队可信度会进入产品判断。Windsurf 创始人离开、团队被收购相关事件，被用户直接翻译成“不再信任这个工具”。Lovable 分析里要保留这个点，因为它说明 founder/team stability 也会影响产品 adoption。

“With these tools, you are trusting the team behind the tool you are building with. When a founder just leaves, what kind of confidence does that instill with users and investors? I know people at Windsurf, but I probably would not use Windsurf again because the trust is gone.” —— Startup Ideas Podcast, 2025-11-03

## Lovable 的差异化是替 non-developer 抽象掉 integration/back-end headache

这条用户 ranking 对 Lovable 的评价很有用：Lovable 和 Bolt 的痛点原来在后端和集成，非开发者需要注册、连接、复制 API keys，这些对开发者 trivial，对普通人是 headache。Lovable 的价值在于把这些东西抽象掉，让用户一启动项目就能更顺滑地拿到后端能力。这个反馈和 Anton 的 simplicity 战略相互印证。

“The pain point with Lovable and Bolt was integrations. You needed a backend, signing up, connecting things, copying API keys. To a developer those are trivial; to a non-developer that is a headache. What Lovable did was abstract a lot of that away, so when you boot up a project, it fetches and connects things for you.” —— Startup Ideas Podcast, 2025-11-03

## Cursor/Windsurf 的主客户仍是 developer，Lovable 的使用者定义更宽

用户访谈里提到 Cursor 团队明确说主目标仍是 developers，非开发者使用很有意思但不会改变核心决策。这个外部反馈有助于区分赛道：Cursor/Claude Code 更像专业开发工具，Lovable 更像 non-developer creation platform。两者功能会交叉，但产品选择、默认复杂度、支持体系和 GTM 都不一样。

“When you talk to Windsurf or Cursor, they tell you they are building for developers. I spoke to someone at Cursor and asked if non-technical people using it changes how they build. He said no: they are building for developers. It is cool to see non-developers use it, but the main demographic is developers.” —— Startup Ideas Podcast, 2025-11-03

## 这类 ranking 虽然主观，但反映了 Lovable 在用户心智里属于 app builder 而非 IDE

这条材料的价值不在排名准确度，而在语言方式：讨论者天然把 Lovable、Bolt、v0 放在 app builder 桶里，把 Cursor、Claude Code、Windsurf 放在 coding tool 桶里。Lovable 如果未来要扩 enterprise/professional workflow，就必须解决这个心智边界：既保持非技术用户的简单性，又让更复杂的团队相信它能接近 production 和 internal workflow。

“When we compare these tools, Lovable sits with Bolt and v0 as the app-builder side, while Cursor, Claude Code, Codex, and Windsurf are the more technical coding side. Lovable is easier for non-developers, but technical users may still run to Claude Code, Codex, or Cursor for deeper work.” —— Startup Ideas Podcast, 2025-11-03
